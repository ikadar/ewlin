## Overview

This release migrates the JCF client field autocomplete from using hardcoded `MOCK_CLIENTS` to an RTK Query endpoint. The `getClientSuggestions` query routes through the hybrid base query, supporting both mock mode (for development/testing) and real API mode (for production).

**Key deliverable:** Client autocomplete fetches suggestions from API with debouncing.

## Technical Notes

### Current Flow (before v0.5.5)

```typescript
// Hardcoded client list
import { MOCK_CLIENTS } from '../../mock/reference-data';

const clientSuggestions = useMemo(
  () => [
    ...MOCK_CLIENTS.map((c) => ({ label: c.name, value: c.name })),
    ...sessionClients.map((s) => ({ label: s, value: s, category: 'nouveau' })),
  ],
  [sessionClients]
);
```

### New Flow (after v0.5.5)

```typescript
// RTK Query with debounce
import { useGetClientSuggestionsQuery } from '../../store';
import { useDebouncedValue } from '../../hooks';

const debouncedQuery = useDebouncedValue(client, 300);
const { data: apiClients = [] } = useGetClientSuggestionsQuery(debouncedQuery, {
  skip: debouncedQuery.length < 2,
});

const clientSuggestions = useMemo(
  () => [
    ...apiClients.map((name) => ({ label: name, value: name })),
    ...sessionClients.map((s) => ({ label: s, value: s, category: 'nouveau' })),
  ],
  [apiClients, sessionClients]
);
```

### API Endpoint

- **Mock mode:** `GET /jobs/clients?q={prefix}`
- **Real mode:** `GET /api/v1/jobs/clients?q={prefix}`

### Response Type

```typescript
// From @flux/types
type ClientSuggestionsResponse = string[];
```

### Debouncing

To avoid excessive API calls, we'll use a 300ms debounce on the input value. The query is skipped if the input is less than 2 characters.
