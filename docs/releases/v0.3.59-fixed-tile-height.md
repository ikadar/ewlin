## Overview

Change the task tile height in the Job Details panel from proportional (based on duration) to a fixed 32px height. This provides a cleaner, more consistent list view where all tasks are equally readable regardless of their duration.

**Key changes:**
- Remove `getHeight()` function that calculates height based on duration
- Set fixed height of 32px for all task tiles
- Ensure content fits within 32px (already using truncate)
- Remove inline `style={{ height: ... }}` in favor of Tailwind class

## Scope

### In Scope

**TaskTile Height Change:**
- Remove duration-based height calculation (`getHeight()` function)
- Replace inline style height with fixed Tailwind class `h-8` (32px)
- Verify content fits (station name + duration info)
- Same height for both scheduled and unscheduled tasks

**Visual Consistency:**
- All tasks in Job Details Panel have same height
- Short tasks (15 min) no longer tiny and hard to read
- Long tasks (4 hours) no longer excessively tall

### Out of Scope

- Grid tile height (remains proportional to duration)
- Task content layout changes (keep existing flex layout)
- Unavailability overlay SVG (v0.3.60)

## Technical Notes

### Current Implementation (Proportional Height)

```typescript
// In TaskTile.tsx
const getHeight = (): number => {
  if (task.type === 'Internal') {
    const totalMinutes = task.duration.setupMinutes + task.duration.runMinutes;
    const height = Math.max(20, Math.round((totalMinutes / 60) * 100));
    return Math.min(height, 200); // Cap at 200px
  }
  return 60; // Default for outsourced
};

// Usage:
<div style={{ height: `${height}px` }}>...</div>
```

### New Implementation (Fixed Height)

```typescript
// Remove getHeight() function entirely
// Replace inline style with Tailwind class

// Scheduled task:
<button className="h-8 pt-0.5 px-2 text-sm border-l-4 ...">

// Unscheduled task:
<div className="h-8 pt-0.5 px-2 text-sm border-l-4 ...">
```
