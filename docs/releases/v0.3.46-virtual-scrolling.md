## Overview

Implement virtual scrolling (windowed rendering) for the multi-day grid to enable performant navigation across many days without DOM bloat. This release fixes the performance issues introduced in v0.3.44, where a 365-day buffer made dragging unusably slow.

**Fixes:** Performance regression from v0.3.44 (DateStrip Redesign)
**Completes:** REQ-09.1 (Infinite scroll) properly

## Scope

### In Scope

**Virtual Scrolling Implementation:**

| Feature | Description |
|---------|-------------|
| **Windowed grid rendering** | Only render ±3 days around focused day (7 days DOM vs 365) |
| **Virtual scroll container** | Full scrollable height maintained, content dynamically rendered |
| **DateStrip windowing** | Only render visible date cells |
| **Scroll position sync** | Grid and DateStrip remain synchronized |
| **Performant dragging** | Drag operations remain smooth with large date ranges |

**Performance Targets:**

| Metric | Current (31 days) | Target (365+ days) |
|--------|-------------------|-------------------|
| DOM elements | ~5000 | ~1000 |
| Drag frame rate | 60 FPS | 60 FPS |
| Initial render | <100ms | <100ms |
| Scroll jank | None | None |

### Out of Scope

- True infinite scroll (dynamically loading dates beyond initial range)
- Server-side date range fetching
- Timeline virtualization (hour rows) - only day-level virtualization

## Technical Notes

### Problem Analysis

**v0.3.44 Implementation:**
```typescript
const DAY_COUNT = 365;
// Results in:
// - Grid: 365 * 24 = 8760 hours rendered
// - Each station column: 8760 hour slots
// - DOM size: ~50,000+ elements
// - Drag performance: <10 FPS (unusable)
```

**Current Workaround (v0.3.45):**
```typescript
const DAY_COUNT = 31; // Reduced for performance - infinite scroll TBD
```

### Solution: Windowed Rendering

**Concept:**
```
Total virtual range: 365 days (or more)
Rendered content: Only ±BUFFER_DAYS around focusedDay

┌─────────────────────────────────────────────────┐
│  Virtual space (no DOM)  │  Day -180 to -4     │
├─────────────────────────────────────────────────┤
│  RENDERED CONTENT        │  Day -3 to +3       │ ← Only this in DOM
│  - Station columns       │  (7 days)           │
│  - Tiles                 │                     │
│  - Grid lines            │                     │
├─────────────────────────────────────────────────┤
│  Virtual space (no DOM)  │  Day +4 to +185     │
└─────────────────────────────────────────────────┘

Scrollable height = TOTAL_DAYS * dayHeightPx (constant)
Rendered content positioned with CSS transform
```

### Implementation Approach

**1. Virtual Scroll Container:**
```typescript
interface VirtualScrollState {
  totalDays: number;           // 365 or configurable
  bufferDays: number;          // 3 (render ±3 days)
  focusedDayIndex: number;     // Calculated from scroll position
  visibleRange: {
    start: number;             // First day to render
    end: number;               // Last day to render
  };
}
```

**2. Scroll Position → Visible Range:**
```typescript
const useVirtualScroll = (scrollTop: number, pixelsPerHour: number) => {
  const dayHeightPx = 24 * pixelsPerHour;
  const focusedDayIndex = Math.floor(scrollTop / dayHeightPx);

  return {
    visibleRange: {
      start: Math.max(0, focusedDayIndex - BUFFER_DAYS),
      end: Math.min(TOTAL_DAYS - 1, focusedDayIndex + BUFFER_DAYS),
    },
    offsetY: visibleRange.start * dayHeightPx,
  };
};
```

**3. Grid Rendering:**
```tsx
<div
  className="virtual-scroll-container"
  style={{ height: TOTAL_DAYS * dayHeightPx }}
>
  <div
    className="rendered-content"
    style={{ transform: `translateY(${offsetY}px)` }}
  >
    {/* Only render days in visibleRange */}
    {visibleDays.map(day => (
      <DaySlice key={day} ... />
    ))}
  </div>
</div>
```

**4. DateStrip Windowing:**
```typescript
// Similar approach - only render visible date cells
const visibleDateCells = dates.slice(visibleRange.start, visibleRange.end + 1);
```
