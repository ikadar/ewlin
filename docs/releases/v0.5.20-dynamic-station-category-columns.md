## Overview

A Flux Dashboard korábban 9 hardkódolt station category oszlopot mutatott (`STATION_CATEGORIES` konstans, `StationCategoryId` union type). Ez a release lecseréli ezeket dinamikus, API-ból érkező adatokra: az oszlopfejlécek a `GET /station-categories` endpointból jönnek, az oszlopsorrendet a `display_order` mező határozza meg.

Emellett bekötjük a valós station state adatokat is: minden element minden station category cellájában a tényleges schedule alapján jelenik meg az állapot (done / late / in-progress / planned / empty), a jelenlegi schedule legrosszabb állapotának aggregálásával.

## Technical Notes

### Station state logika

```
Task state (per task):
  1. task.status === Completed           → 'done'
  2. Nincs assignment a schedule-ben     → 'empty'
  3. assignment.isCompleted === true     → 'done'
  4. assignment.scheduledEnd < now       → 'late'
  5. assignment.scheduledStart <= now    → 'in-progress'
  6. assignment.scheduledStart > now     → 'planned'

Aggregálás (per element + categoryId):
  Worst state: late=0 > in-progress=1 > planned=2 > done=3 > empty=4
```

### StationCategoryId eltávolítása

A `FluxTable.tsx` korábban `el0.stations[cat.id as StationCategoryId]` pattern-t használt. Átírva `Record<string, FluxStationData>` alapúra, type assertion eltávolítva.

### Abbreviation fallback

DB-ben üres abbreviation esetén frontend fallback: `cat.abbreviation || cat.name.substring(0, 5)`. A `Version20260302200000.php` migration beállítja a pontos rövidítéseket a mockup.html alapján.
