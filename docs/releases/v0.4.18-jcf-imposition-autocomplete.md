## Overview

Replaces the plain text input for the "Imposition" field in the JCF Elements Table with a two-step autocomplete component. Users first select a sheet format (e.g., "50x70"), then a poses count (e.g., "8"). The stored DSL format is `LxH(poses)` (e.g., `50x70(8)`), while the pretty display shows `LxHcm Nposes/f` (e.g., `50x70cm 8poses/f`).

## Technical Notes

- **Two-step autocomplete**: Similar to Papier (v0.4.17). When the user selects a sheet format, `(` is appended and suggestions switch to poses for that format. When poses is selected, `)` is appended to complete the DSL.
- **DSL parsing**: The opening parenthesis `(` is the separator. `indexOf('(')` splits format from poses. The closing `)` is stripped during editing and re-added on completion.
- **Poses storage**: `FeuilleFormat.poses` is `number[]` in `@flux/types`. Displayed as strings in suggestions.
- **Session learning**: `learnFeuilleFormat` uses poses-merge strategy -- if the format exists, new poses are merged into the existing poses array. If not, a new entry is added.
- **Standalone component**: Following the JcfPapierAutocomplete pattern -- standalone component reusing shared utilities (`useLazyLoadSuggestions`, `highlightMatch`, `mergeWithSession`).
- **Pretty format**: `50x70cm 8poses/f` -- adds `cm` after dimensions and `poses/f` after count.
