## Overview

Correction du bug de désynchronisation entre le ghost tile et le message de validation en mode Pick & Place. Le message "Hors horaires (HH:MM)" affichait une heure décalée par rapport à la position réelle du curseur.

**Origine:** Party Mode discussion (2026-01-15) - rapport de bug sur l'offset du feedback live.

---

## Prerequisites

- [x] v0.3.59 Pick & Place Feature Parity released

## Scope

### In Scope

- Synchronisation du message de validation avec la position réelle du ghost
- Génération du message dans la boucle RAF (60fps)
- Remplacement de l'heure stale par l'heure calculée en temps réel

### Out of Scope

- Modifications du système de validation
- Changements au throttle de validation (conservé à 100ms)
- Autres types de messages de validation

---

## Technical Notes

### Problème identifié

```
AVANT (désynchronisé):
├─ Ghost position: RAF (60fps, temps réel)
├─ Validation: Throttled 100ms
└─ Message: Généré lors de la validation → STALE

Résultat: "Hors horaires (14:30)" alors que le ghost est à 14:45
```

### Solution implémentée

```
APRÈS (synchronisé):
├─ Ghost position: RAF (60fps, temps réel)
├─ Validation: Throttled 100ms → fournit les conflicts
└─ Message: Généré dans RAF loop avec temps réel

Résultat: "Hors horaires (14:45)" synchronisé avec le ghost
```

### Architecture

```typescript
// App.tsx: Passe conflicts au lieu de message
setPickValidation({
  isValid,
  hasPrecedenceConflict,
  suggestedStart,
  hasWarningOnly,
  conflicts: validationResult.conflicts,  // ← nouveau
});

// PickPreview.tsx: Génère message dans RAF loop
const currentTime = yPositionToTime(snappedY, startHour, gridStartDate, pixelsPerHour);
const message = generateRealTimeMessage(conflicts, isValid, hasWarningOnly, currentTime);
```
