## Overview

Replaces the plain text input for the "Precedences" field in the JCF Elements Table with an autocomplete component that suggests other element names. Users can select one or multiple predecessors as a comma-separated list (e.g., "INT,COUV"). The component excludes the current element from suggestions (preventing self-reference) and excludes already-selected predecessors.

Additionally, the ElementsTable gains cascading update logic: when an element is renamed, all precedences referencing the old name are updated; when an element is removed, it is removed from all precedences.

## Technical Notes

- **No DSL/pretty conversion**: Unlike Papier/Imposition, precedences are plain comma-separated element names. No display transformation needed.
- **Multi-value parsing**: Uses comma as separator. `getFilterPart(value)` extracts text after the last comma for filtering. `getSelectedElements(value)` splits and trims all committed parts.
- **Selection logic**: When a name is selected, replaces the partial text after the last comma with the full name. Committed names (before commas) are preserved.
- **Self-reference prevention**: The current element's name is excluded from suggestions.
- **Already-selected exclusion**: Elements already in the comma list are excluded from suggestions.
- **Cascading updates**: When renaming an element, `precedences` fields of ALL elements are scanned and old names replaced. When removing an element, its name is removed from all `precedences` fields.
- **No session learning**: Suggestions come solely from the current elements list.
- **Simpler component**: This wraps `JcfAutocomplete` concept but with string[] suggestions (not Suggestion objects). Can be standalone for clarity.
