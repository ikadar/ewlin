## Overview

Replaces the plain textarea for the "Sequence" field in the JCF Elements Table with a multi-line autocomplete component. Each line represents a production step. In Poste mode, the component suggests machine names (G37, 754, GTO, Stahl, etc.) from the preset list, then prompts for duration input in the format `PosteName(duration)` or `PosteName(setup+run)`. Completed lines are validated per-line with smart validation (incomplete lines are not flagged).

This is the first of three Sequence releases (v0.4.20 Poste mode, v0.4.21 ST mode, v0.4.22 Workflow suggestions).

## Scope

### In Scope

- Multi-line textarea with per-line autocomplete (portal dropdown)
- Poste mode: suggest machine names from `POSTE_PRESETS` (16 entries)
- Duration input after poste selection: `PosteName(duration)` or `PosteName(setup+run)`
- "ST:" option shown in poste suggestions (entry point for v0.4.21)
- Per-line validation regex: `/^[A-Za-z0-9_]+\(\d+(\+\d+)?\)$/`
- Smart validation: only validate lines with closing paren (incomplete lines not flagged)
- Dropdown positioned at cursor within textarea (portal-based)
- Keyboard navigation (ArrowUp/Down, Enter, Escape)
- Table navigation delegation (Tab, Shift+Tab, Alt+Arrow)
- Session learning for postes (`learnPoste`)

- Unit tests (DSL utilities + component)

- ST mode full implementation (v0.4.21)
- Workflow-guided suggestions with priority/star markers (v0.4.22)
- Per-line error highlighting/red background (v0.4.23 -- validation)
- Duration validation (any digits accepted for now)

## Technical Notes

- **Multi-line textarea**: Unlike other autocompletes (single `<input>`), this is a `<textarea>` with per-line parsing. Each line is independently autocompleted.
- **Cursor-relative dropdown**: Dropdown is rendered as a portal (`createPortal`) positioned at the current cursor position within the textarea. Uses a mirror div technique to calculate cursor pixel coordinates.
- **State machine per line**: `parseLine(input)` determines the current step: `poste` (machine name), `duration` (after `(`), or `complete` (has `(...)`).
- **Two-step per line**: Step 1 = select poste name (appends `(`), Step 2 = type duration and close with `)` or select from dropdown.
- **Duration suggestions**: Default durations `['20', '30', '40', '60', '20+30', '20+40', '30+60']` shown when in duration step.
- **"ST:" entry point**: At poste step, "ST:" appears as last suggestion. Selecting it inserts `ST:` and opens v0.4.21 flow (not yet implemented -- selecting "ST:" just inserts the text).
- **Session learning**: When a line is completed (has `(...)`), the poste name is learned via `learnPoste`. Session postes appear first in suggestions.
- **Smart validation**: `isSequenceLineComplete(line)` checks for closing `)`. Only complete lines are validated against the regex. This prevents false errors while typing.
- **No DSL/pretty conversion**: Unlike papier/imposition, sequence values are stored exactly as typed.
