## Overview

Two-part release to scale the base app to ~80% of original size:
1. **Font-size standardization**: Set root font-size to 13px (81.25% of 16px default), convert explicit px font sizes to standard Tailwind classes
2. **Grid scaling**: Scale all zoom levels to 80% (e.g., 100% zoom now uses 64px/hour instead of 80px/hour)

The app architecture is well-suited for grid scaling — the `pixelsPerHour` parameter already controls most sizing. Only 2 files have hardcoded values that need fixing.

## Scope

### In Scope

**Part 1: Font-size Standardization**
- Set root font-size to 13px (reference/jcf compatible)
- Convert `text-[11px]` → `text-sm` (11.375px with 13px root)
- Convert `text-[13px]` → `text-base` (13px with 13px root)
- Convert `text-[10px]` → `text-xs` (9.75px with 13px root)
- Keep edge cases: `text-[6px]`, `text-[9px]`, `text-[15px]` (no good Tailwind equivalent)

**Part 2: Grid Scaling (All Zoom Levels)**
- Scale all zoom levels to 80% of original values
- 100% zoom now uses 64px/hour (was 80px/hour)
- Fix hardcoded height calculations in DragPreview and PickPreview
- Fix scroll position calculations to use root font-size aware values

### Out of Scope

- CSS zoom or transform approaches (not needed)
- Column width scaling (optional, can be added later)

## Technical Analysis

### Architecture Assessment

The codebase is **well-architected for scaling**:

| Aspect | Status | Notes |
|--------|--------|-------|
| Snap grid | ✅ Safe | `snapToGrid(y, pixelsPerHour)` — parametrized |
| Constraint lines | ✅ Safe | `timeToYPosition()` — dynamic |
| Mouse coordinates | ✅ Safe | `getBoundingClientRect()` relative — zoom-agnostic |
| Tile positioning | ✅ Safe | Using `timeToYPosition()` — scales correctly |
| Virtual scroll | ✅ Safe | Uses `dayHeightPx` parameter |
| Tile heights | 🔴 Critical | 2 hardcoded `* 80` multipliers |
| Column widths | 🟡 Optional | Fixed Tailwind classes don't scale |
