## Overview

A második lépés a Production Flow Dashboard megvalósításában: interaktív szűrés, URL-alapú tab navigáció és billentyűzetes vezérlés. A v0.5.15-ben felépített read-only táblázatra rákerül a keresősáv, az 5-tabos szűrő, és a teljes keyboard shortcut rendszer. A tab állapota nested route-ban él (`/flux`, `/flux/prepresse`, stb.), így böngészőfrissítés után megmarad.

## Scope

### In Scope

- Nested routes: `/flux` (Tous), `/flux/prepresse`, `/flux/papier`, `/flux/formes`, `/flux/plaques`
- Tab bar: 5 tab aktív/inaktív vizuális állapottal, dinamikus count badge-ekkel
- Count badge-ek: tab szűrő + aktív keresés kombinációja alapján számolódik
- Full-text keresés: real-time, case-insensitive, összes szöveges oszlopra (ID, Client, Designation, Transporteur, prerequisite badge label)
- Toolbar: "Flux de production" cím, "Nouveau job" gomb (`Alt+N` hint badge), keresősáv
- Keyboard shortcuts: `Alt+←/→` (tab váltás), `Alt+↑/↓` (sor navigáció), `Alt+F` (keresőmező fókusz), `Alt+N` (új job → `/job/new`)

- Filterezés multi-element sorokra az aggregált worst értékek alapján (a parent sor aggregált badge-ét vizsgálja, nem az egyes elemekét)

- Prerequisite listbox dropdown-ok (v0.5.17)
- Multi-element expand/collapse animáció (v0.5.17)
- Delete/Edit akciók (v0.5.17)
- Parti toggle interaktivitás (v0.5.17)
- Station oszlopok rendezhetősége (halasztva — qa.md K9.2)
- Alt+↑/↓ valódi DOM focus kezelés (v0.5.17-ben DOM focus a listbox-okhoz szükséges; v0.5.16-ban visual highlight elegendő)
- API integráció (v0.5.18)

## Technical Notes

- **Nested routes:** A `routes.tsx`-ben a `/flux` route-hoz nested route-ok kerülnek (index + 4 child). A `FluxPage` a `useLocation`-ból olvassa az aktív tab-ot.
- **Tab ↔ route mapping:** `'/flux' → 'all'`, `'/flux/prepresse' → 'prepresse'`, stb. Navigáció `useNavigate()`-tel.
- **Search state:** `FluxPage`-en belül `useState`, nem URL-ben (spec szerint: tab az URL-ben, keresés nem).
- **Count badge számítás:** Minden tab countja = a jobList szűrve (tab filter AND search) darabszáma. A `counts` objektum minden tab-hoz tartalmaz egy értéket, mindig az összes ötre számolva.
- **Filtering:** `FluxPage` szűri a `jobs` tömböt tab + search alapján, és a szűrt tömböt adja át a `FluxTable`-nek. A `FluxTable` belső rendezése (ID asc) a szűrt tömbön működik.
- **Search scope:** A parent sor szöveges mezői: `id`, `client`, `designation`, `transporteur`. Prerequisite badge label-ek: a parent soron megjelenített (worst) értékek — `bat`, `papier`, `formes`, `plaques` stringek. Sub-row szövegek NEM kerülnek keresésre (a spec sub-row visibility-t a parent filter-eredményéhez köti).
- **Keyboard shortcuts:** `useEffect` + `document.addEventListener('keydown')`. `Alt+F`: `searchRef.current?.focus()`. `Alt+N`: `navigate('/job/new')`. `Alt+←/→`: tab ciklus. `Alt+↑/↓`: `focusedRowIndex` state, scroll into view.
- **Alt+↑/↓ implementáció:** A FluxPage tartja a `focusedRowIndex`-et (visible parent rows indexébe), és egy `data-flux-row-focused` attribútumot set-el a megfelelő sorra. CSS kiemeli a fókuszált sort (indigo háttér + outline). v0.5.17-ben ez a DOM focus-ra lesz kötve a listbox interakciókhoz.

### Filtering Logic

- [ ] **fluxFilters.ts — szűrő függvények**
  - Description: `filterByTab(job, tab)` — spec 6.4 filter functions. `filterBySearch(job, search)` — case-insensitive substring keresés: id, client, designation, transporteur, + bat/papier/formes/plaques string értékek. `computeTabCounts(jobs, search)` — returns `Record<TabId, number>`. `TabId` = `'all' | 'prepresse' | 'papier' | 'formes' | 'plaques'`. A filterByTab multi-element soroknál az aggregált worst értékeket használja (a `FluxJob` parent row mezői már az aggregált worst értékeket tartalmazzák — ez a v0.5.15-ös `fluxStaticData.ts`-ben is így van).
  - Files: `src/components/FluxTable/fluxFilters.ts` (új)
  - Tests: `src/components/FluxTable/fluxFilters.test.ts` (új) — spec 6.5 verification matrix összes kombinációja, search edge case-ek

### FluxPage Integration

- [ ] **FluxPage újraszabása**
  - Description: Integrálja a FluxToolbar-t és FluxTabBar-t. `useLocation` alapján meghatározza az aktív tab-ot. `useState` a keresőmező értékéhez. `useMemo` a filtered jobs + tab counts számításához. A szűrt jobs-t adja át a FluxTable-nek. Layout: `flex-col`, toolbar (border-b), tab bar (border-b), table (flex-1 overflow).
  - Files: `src/pages/FluxPage.tsx`
  - Tests: `src/pages/FluxPage.test.tsx` (módosítás)

### Keyboard Shortcuts

- [ ] **Keyboard shortcut kezelő**
  - Description: `useEffect` + `document.addEventListener('keydown', handler)` a `FluxPage`-ben. `Alt+F`: keresőmező fókuszálása (`searchRef`). `Alt+N`: `navigate('/job/new')`. `Alt+←`: előző tab (ciklikus). `Alt+→`: következő tab (ciklikus). `Alt+↑`: `focusedRowIndex--` (visible rows közt, clamp 0-nál). `Alt+↓`: `focusedRowIndex++` (visible rows közt). Minden shortcut: `e.preventDefault()` az alapértelmezett böngésző viselkedés megakadályozásához.
  - Files: `src/pages/FluxPage.tsx`
  - Tests: E2E: `flux-navigation.spec.ts`

### Row Focus (Alt+↑/↓)

- [ ] **Fókuszált sor vizuális kiemelése**
  - Description: A `FluxTable` props-ot kap: `focusedJobId?: string`. Ha meg van adva, a megfelelő `<tr>` kap egy `data-flux-focused="true"` attribútumot és CSS kiemelést (halvány indigo háttér + outline). A FluxPage kezeli a `focusedJobId` state-et (visible sorted rows alapján indexeli).
  - Files: `src/components/FluxTable/FluxTable.tsx`, `src/pages/FluxPage.tsx`
  - Tests: E2E: `flux-navigation.spec.ts`
