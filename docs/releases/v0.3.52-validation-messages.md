## Overview

Display human-readable validation messages during drag operations. When a tile cannot be placed at a position (red border), users need to understand WHY the drop is invalid. This release adds a validation message tooltip that appears near the drag preview showing the specific conflict reason.

**Implements:** [REQ-07](../ux-ui/tmp/refactored-new-requirements-03-en.md#req-07-human-readable-validation-messages)

## Scope

### In Scope

- **Validation message display** during drag operations
- **Message tooltip/badge** near the drag preview
- **Message types** for all conflict categories:
  - Station unavailable (availability conflict)
  - Task conflict (station booking)
  - Precedence conflict (too early/too late)
  - BAT not approved (approval gate)
  - Group capacity exceeded
  - Station mismatch
  - Deadline conflict
- **French message localization** (messages in French for consistency with UI)
- **Primary message display** (show most important conflict first)

### Out of Scope

- Message history/log panel
- Custom message formatting options
- Warning-level messages (only show for blocking conflicts)
- Multiple message display (show only primary)

## Technical Notes

### Message Display Location

The validation message will appear as a tooltip below the drag preview tile:

```
        ┌──────────────┐
        │   Dragged    │
        │    Tile      │
        └──────────────┘
        ┌──────────────────────────┐
        │ ⚠ Station fermée         │  ← Validation message
        │   12:00-13:00 (pause)    │
        └──────────────────────────┘
```

### Message Sources

Messages come from `packages/validator/src/validators/`:

| Validator | Conflict Type | Example Message |
|-----------|---------------|-----------------|
| availability.ts | AvailabilityConflict | "Station fermée 12:00-13:00" |
| station.ts | StationConflict | "Conflit avec tâche existante" |
| precedence.ts | PrecedenceConflict | "Prédécesseur se termine à 14:00" |
| approval.ts | ApprovalGateConflict | "BAT non approuvé" |
| group.ts | GroupCapacityConflict | "Capacité groupe dépassée (3/2)" |
| stationMatch.ts | StationMismatchConflict | "Station incompatible" |
| deadline.ts | DeadlineConflict | "Échéance dépassée" |

### Implementation Approach

1. **Extract primary message** from validation result in `DragPreview`
2. **Create ValidationMessage component** for styled tooltip
3. **Show message only for blocking conflicts** (red border states)
4. **Position below drag preview** using absolute positioning

```typescript
// In DragPreview.tsx
const primaryMessage = validation.conflicts?.[0]?.message;
const showMessage = !validation.isValid && primaryMessage;
```
