## Overview

Replaces the plain text input for the "Papier" field in the JCF Elements Table with a two-step autocomplete component. Users first select a paper type (e.g., "Couché mat"), then a grammage (e.g., "135"). The stored DSL format is `Type:Grammage` (e.g., `Couché mat:135`), while the pretty display shows `Type Grammageg` (e.g., `Couché mat 135g`).

## Technical Notes

- **Two-step autocomplete**: Unlike single-step autocompletes (Impression, Surfacage), Papier requires a two-step flow. When the user selects a type, `:` is appended and suggestions switch to grammages for that type.
- **DSL parsing**: The colon (`:`) is the separator. `lastIndexOf(':')` is used to handle type names that might contain special characters.
- **Grammage normalization**: Grammages are stored as numbers in `PaperType.grammages[]` but displayed with `g` suffix in pretty format.
- **Session learning compound merge**: `learnPapier` uses replace-merge strategy — if the type exists, the full entry is replaced (allowing grammage list extension). The component must build the merged grammage list before calling `learnPapier`.
- **Component wraps JcfAutocomplete**: Unlike the reference implementation which is standalone, the main app implementation wraps `JcfAutocomplete` for consistency. The component manages its own suggestion list that switches between types and grammages based on input state.
