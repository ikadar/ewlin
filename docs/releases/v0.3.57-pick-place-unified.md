## Overview

Unification du modèle d'interaction : les tiles planifiées sur la grille deviennent pickables par simple clic, comme les tâches de la sidebar. Cela crée une expérience cohérente et simplifie le modèle mental de l'utilisateur. Un bouton info (i) permet d'accéder aux détails du job sans interférer avec le pick.

**Origine:** Party Mode discussion (2026-01-15) - unification UX drag & drop vers Pick & Place.

## Scope

### In Scope

- Tiles planifiées pickables par clic (zone entière hors bouton info)
- Bouton info (i) sur les tiles grid (translucide 30%, opacity 100% au hover)
- Curseur `grab` au survol des tiles pickables
- État "picked" sur la tile source (style planifié + pulse)
- Tiles complétées non pickables
- Escape pour annuler et restaurer la position
- Scroll automatique vers la colonne cible
- Fade out des autres colonnes pendant le pick

### Out of Scope

- Suppression du drag & drop existant (transition progressive)
- Multi-sélection de tiles (release ultérieure)
- Pick sur tiles outsourced

## Technical Notes

### Interaction Model

```
GRID TILE (planifiée, non complétée)
┌────────────────────────────────┐
│ Impression    14:30         i │
└────────────────────────────────┘
     │                        │
     │ clic                   │ clic
     ▼                        ▼
   PICK                    DÉTAILS
   (mode placement)        (sidebar job)

SIDEBAR TILE (non planifiée)
┌─────────────────────┐
│ Impression          │
│ Durée: 2h           │  clic = PICK (inchangé)
└─────────────────────┘
```

### États des tiles grid

| État | Fond | Border | Curseur | Bouton i | Pickable |
|------|------|--------|---------|----------|----------|
| Planifiée | Couleur job | 4px left | `grab` | Oui (30% → 100%) | Oui |
| Planifiée (hover) | Couleur job | 4px left | `grab` | Oui (100%) | Oui |
| Picked | Couleur job | 4px left | - | Non | - |
| Complétée | Gris | Aucune | `default` | Oui (pour détails) | Non |

### Bouton Info (i)

```css
.tile-info-button {
  opacity: 0.3;
  transition: opacity 150ms ease-out;
}

.task-tile:hover .tile-info-button {
  opacity: 1;
}

.tile-info-button:hover {
  cursor: pointer; /* vs grab pour le reste de la tile */
}
```

### Pick depuis Grid vs Sidebar

```typescript
interface PickSource {
  type: 'sidebar' | 'grid';
  originalAssignment?: TaskAssignment; // Pour grid: l'assignment existant
}

// Au pick depuis grid:
// 1. Stocker l'assignment original
// 2. Afficher la tile en état "picked" (pas la supprimer)
// 3. Au place: mettre à jour l'assignment (reschedule)
// 4. Au cancel: restaurer visuellement (rien à faire côté data)
```

### Séquence Pick depuis Grid

```
1. PICK (clic sur tile grid)
   │
   ├─► Immédiat:
   │   - Tile passe en état "picked" (pulse opacity)
   │   - Stocker originalAssignment dans pickState
   │   - Curseur body → grabbing
   │
   ├─► Animation 300ms:
   │   - Scroll vers colonne cible (même station)
   │   - Fade out autres colonnes
   │
   └─► Placement/Cancel:
       - PLACE: updateAssignment (reschedule)
       - CANCEL: tile reprend état normal (data inchangée)
```

### Summary

Les tiles planifiées sur la grille sont maintenant pickables par simple clic, unifiant le modèle d'interaction avec la sidebar. Un bouton info permet d'accéder aux détails sans interférer avec le placement.

- Nouveau type `PickSource` pour distinguer l'origine du pick
- Stockage de l'assignment original pour reschedule
- Réutilisation de la logique visuelle v0.3.55 (scroll, fade, validation)
- Drag & drop existant maintenu (transition progressive)

### Migration Notes

Aucune migration requise. Le drag & drop reste fonctionnel en parallèle.
