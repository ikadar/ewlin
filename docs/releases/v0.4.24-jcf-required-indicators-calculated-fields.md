## Overview

Adds Level 2 validation (required field indicators) and qteFeuilles auto-calculation to the JCF elements table. Required-but-empty fields show an amber dot indicator. The qteFeuilles field auto-calculates based on job quantity, element quantity, and imposition poses, with a toggle to switch between auto and manual modes.

This is the second of three validation releases (v0.4.23 Level 1 Live, v0.4.24 Level 2 Required, v0.4.25 Level 3 Submit).

## Scope

### In Scope

- **Required field indicators (Level 2)**:
  - Amber dot indicator for required-but-empty fields
  - `sequence` always required if element has content (job mode only, not template)
  - BLOC SUPPORT trigger: if any of (imposition, impression, surfacage, format) → papier, pagination, format, qteFeuilles, imposition required
  - BLOC IMPRESSION trigger: if any of (imposition, impression) → impression required
  - Template mode has no required fields
- **qteFeuilles auto-calculation**:
  - Formula: `ceil((jobQuantity × elementQuantity) / poses)`
  - Extract poses from imposition DSL
  - Auto/manual toggle per element (Calculator icon button)
  - Auto mode: green text + green icon
  - Manual mode: normal text + gray icon
  - Typing in field switches to manual mode
  - Toggle button to switch back to auto
- **Recalculation triggers**:
  - jobQuantity changes
  - elementQuantity changes
  - imposition changes (affects poses)
- **Unit tests** for required field logic and calculation
- **E2E tests** for indicators and calculation behavior

### Out of Scope

- Level 3 validation (submit blocking) — v0.4.25
- Error summary display
- Template mode specific features

---

## Technical Notes

- **Required field state**: Use `useMemo` to compute required fields set for O(1) lookup
- **Auto-calculation state**: `autoCalculated: Record<number, boolean>` per element index
- **Indicator visibility**: Show amber dot only when field is required AND empty AND not in error state
- **Poses extraction**: Parse from imposition DSL pattern `(Np?)` or pretty format `N poses`
- **jobQuantity**: Passed as prop from parent (JcfJobHeader)
- **Recalculation**: useEffect that runs when jobQuantity, element quantite, or imposition changes
