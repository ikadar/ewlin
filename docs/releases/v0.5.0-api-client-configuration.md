## Overview

This release configures RTK Query to support both mock mode (for development/testing) and real API mode (for production), while preserving the critical fixture-based testing capability. This is the foundation for all subsequent API integration work in Milestone 5.

**Key deliverable:** A "base query with fixture support" pattern that allows seamless switching between mock data and real API calls based on URL parameters or environment variables.

## Technical Notes

### Base Query Pattern

The solution uses a "base query with fixture support" pattern:

```typescript
const baseQueryWithFixtureSupport = async (args, api, extraOptions) => {
  const fixture = new URLSearchParams(window.location.search).get('fixture');

  if (fixture || import.meta.env.VITE_USE_MOCK === 'true') {
    // Fixture/mock mode: use mock adapter
    return mockBaseQuery(args, api, extraOptions);
  }

  // Production mode: use real API
  return realBaseQuery(args, api, extraOptions);
};
```

### Mock Adapter

The mock adapter translates RTK Query args to existing mock function calls:

```typescript
const mockBaseQuery = async (args) => {
  const { url, method, body } = args;

  // Route to appropriate mock handler
  if (url === '/schedule/snapshot' && method === 'GET') {
    return { data: getSnapshot() };
  }
  // ... other routes
};
```

### Environment Variables

| Variable | Development | Production |
|----------|-------------|------------|
| `VITE_USE_MOCK` | `true` | `false` |
| `VITE_API_URL` | (not needed) | `https://api.flux.example.com` |
