## Overview

This release implements consistent error handling across all API calls in the frontend. It provides user-friendly error feedback through ErrorBoundary components, toast notifications, and inline form validation messages. Network errors and 503 Service Unavailable responses are handled gracefully with retry UI.

**Key deliverable:** Consistent, user-friendly error handling for all API operations.

- [x] `v0.5.0` API Client Configuration (hybrid base query)

## Scope

### In Scope

- ErrorBoundary component for React error catching
- Redux error slice for global error state
- Toast notifications for transient errors (mutations)
- Inline error messages for form validation (JCF - existing)
- Network error detection with retry UI
- 503 Service Unavailable handling with maintenance message

### Out of Scope

- Retry logic with exponential backoff (v0.5.9)
- Error logging to external service (future)
- Offline mode support (future)

## Technical Notes

### Existing Infrastructure

The following components already exist and are leveraged:

| Component | Location | Description |
|-----------|----------|-------------|

| `errorNormalization.ts` | `store/api/errorNormalization.ts` | Error utilities |

### New Components

| Component | Description |
|-----------|-------------|
| `ErrorBoundary` | React error boundary for catching render errors |
| `errorSlice` | Redux slice for global error state |
| `GlobalToast` | Global toast connected to Redux error state |
| `MaintenanceState` | 503 Service Unavailable maintenance page |

### Error Categories

| Category | Handling | UI Element |
|----------|----------|------------|
| Network error (FETCH_ERROR) | Show toast with retry | Toast + ErrorState |
| 503 Service Unavailable | Show maintenance message | MaintenanceState |
| 409 Conflict (validation) | Show inline error | Form error message (existing) |
| 4xx Client errors | Show toast | GlobalToast |
| 5xx Server errors | Show toast with retry | GlobalToast + ErrorState |
| React render errors | Show ErrorBoundary fallback | ErrorBoundary |
