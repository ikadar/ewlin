## Overview

This release migrates the JCF (Job Creation Form) from using direct `api/jobApi.ts` calls to RTK Query mutations. The `createJob` mutation routes through the hybrid base query, supporting both mock mode (for development/testing) and real API mode (for production).

**Key deliverable:** JCF form submission uses RTK Query mutation with proper mock handling and error display.

## Technical Notes

### Current Flow (before v0.5.4)

```typescript
// Direct API call from api/jobApi.ts
import { createJob, transformJcfToRequest } from './api';

const handleSaveJob = async () => {
  const request = transformJcfToRequest(...);
  await createJob(request);  // Uses own mock mode detection
  // Manual state reset and navigation
};
```

### New Flow (after v0.5.4)

```typescript
// RTK Query mutation
import { useCreateJobMutation } from './store';

const [createJob] = useCreateJobMutation();

const handleSaveJob = async () => {
  const request = transformJcfToRequest(...);
  try {
    await createJob(request).unwrap();
    // Cache invalidation is automatic via invalidatesTags: ['Snapshot']
    navigate('/');
  } catch (error) {
    showToast(getErrorMessage(error));
  }
};
```

### Mock Handler Implementation

The mock handler in `mockBaseQuery.ts` will:
1. Parse the `CreateJobRequest`
2. Generate new Job, Element(s), and Task(s) entities
3. Update the mock snapshot state
4. Return a `CreateJobResponse` with the created job data

### Response Type

```typescript
interface CreateJobResponse {
  id: string;
  reference: string;
  client: string;
  description: string;
  workshopExitDate: string;
  status: string;
  elementIds: string[];
  taskIds: string[];
  createdAt: string;
}
```
