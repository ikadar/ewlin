## Overview

This release connects the drag-drop scheduling operations to the RTK Query mutations. Currently, assignment operations use direct `updateSnapshot()` calls. This release migrates them to use the RTK Query mutations (`assignTask`, `rescheduleTask`, `unassignTask`) which route through the hybrid base query (mock or real API).

**Key deliverable:** All assignment CRUD operations use RTK Query mutations with proper error handling.

## Scope

### In Scope

- Replace `updateSnapshot()` calls with RTK Query mutations for:
  - New assignment (sidebar → grid drag, quick placement)
  - Reschedule (grid tile move)
  - Unassign/Recall (double-click, context menu)
- Error handling for mutation failures
- Toast notification for validation errors (409 Conflict)
- Handle `suggestedStart` from backend validation errors

### Out of Scope

- Completion toggle (v0.5.3)
- Optimistic updates (v0.5.8)
- Swap operations (keep using local state for now - not a core API operation)
- Compact operations (keep using local state - not a core API operation)

## Technical Notes

### Current Flow (before v0.5.2)

```typescript
// Direct mock state update
updateSnapshot((snapshot) => ({
  ...snapshot,
  assignments: [...updatedAssignments, newAssignment],
}));
invalidateSnapshot(); // Sync RTK Query cache
```

### New Flow (after v0.5.2)

```typescript
// RTK Query mutation
const [assignTask] = useAssignTaskMutation();

await assignTask({
  taskId: task.id,
  body: {
    targetId: stationId,
    scheduledStart,
    isOutsourced: false,
  },
});
// Cache invalidation is automatic via invalidatesTags: ['Snapshot']
```

### Error Response Structure (409 Conflict)

```typescript
interface ValidationErrorResponse {
  error: 'ValidationFailed';
  message: string;
  conflicts: Array<{
    type: string;
    message: string;
    taskId?: string;
    relatedTaskId?: string;
  }>;
  suggestedStart?: string;
}
```

### Toast Notification

A simple toast component will show validation errors. French text for user-facing messages.

## v0.5.2 – Assignment Operations

**Release Date:** 2026-02-03

### Summary

Migrates assignment operations (create, reschedule, unassign) from direct mock state updates to RTK Query mutations. This prepares the app for real backend API integration.

### What's New

- **RTK Query Mutations**: All assignment operations now use proper API calls
- **Toast Notifications**: Error messages shown via toast component
- **Error Handling**: 409 Conflict errors display validation messages

### Technical Notes

In mock mode, the behavior is identical. In real API mode, operations will call the backend endpoints:
- `POST /tasks/{taskId}/assign` - Create assignment
- `PUT /tasks/{taskId}/assign` - Reschedule assignment
- `DELETE /tasks/{taskId}/assign` - Unassign/Recall
