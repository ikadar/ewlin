# Release v0.4.31 – Sequence Autocomplete: Template-Free Mode

> **Status:** ✅ Released
>
> **Milestone:** M4
>
> **Release Date:** 2026-01-30
>
> **Git Tag:** `v0.4.31`

---

## Overview

Fix the Sequence Autocomplete behavior when no template is selected. Currently, the sequence input doesn't have access to template workflow data (only uses test fixtures). This release:
1. Adds `workflow` field to template data
2. Connects template selection to sequence workflow
3. Implements proper Template-Free Mode (no `★` markers, all postes shown equally)

## Release Notes Draft

```markdown
## v0.4.31 – Sequence Autocomplete: Template-Free Mode

**Release Date:** TBD

### Summary

Fixes the Sequence Autocomplete to properly handle template-free mode. When no template is selected, all postes are shown equally without workflow-based priority markers (★). When a template is selected, the workflow guides suggestions with priority indicators.

### What's New

**Template-Free Mode:**
- No ★ markers when no template selected
- All postes shown with equal priority
- Clean experience for ad-hoc jobs

**Template Mode:**
- Template selection activates workflow
- ★ markers indicate expected next step
- Workflow progresses as operations are added

### Technical

- Templates now include `workflow?: string[]` field
- `sequenceWorkflow` state tracks active workflow
- Dynamic workflow from template selection replaces hardcoded test data
```
