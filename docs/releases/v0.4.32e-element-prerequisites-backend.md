## Release Notes

```markdown
## v0.4.32e – Element Prerequisites Backend API

**Release Date:** 2026-01-31

### Summary

Backend API support for element-level prerequisite tracking. Completes the v0.4.32 series.

### What's New

**New Enum Types:**
- PaperStatus (none, in_stock, to_order, ordered, delivered)
- BatStatus (none, waiting_files, files_received, bat_sent, bat_approved)
- PlateStatus (none, to_make, ready)
- FormeStatus (none, in_stock, to_order, ordered, delivered)

**Element Entity:**
- 4 status fields: paperStatus, batStatus, plateStatus, formeStatus
- 7 date tracking fields: paperOrderedAt, paperDeliveredAt, filesReceivedAt, batSentAt, batApprovedAt, formeOrderedAt, formeDeliveredAt
- isBlocked() method for blocking logic

**New Endpoint:**
- PUT /api/v1/elements/{id}/prerequisites

**Domain Events:**
- ElementPaperStatusUpdated
- ElementBatStatusUpdated
- ElementPlateStatusUpdated
- ElementFormeStatusUpdated

### Breaking Changes

- Removed: PUT /api/v1/jobs/{id}/proof
- Removed: PUT /api/v1/jobs/{id}/plates
- Removed: PUT /api/v1/jobs/{id}/paper
- Job entity no longer has: proofSentAt, proofApprovedAt, platesStatus, paperPurchaseStatus, paperOrderedAt
```
