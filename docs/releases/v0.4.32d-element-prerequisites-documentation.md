## Overview

Documentation updates for element-level prerequisites. This release documents the changes made in v0.4.32a–v0.4.32c across all architecture and domain model files.

**Key changes to document:**
- Prerequisite fields moved from Job to Element level (paperStatus, batStatus, plateStatus, formeStatus)
- Date tracking fields on Element (paperOrderedAt, paperDeliveredAt, filesReceivedAt, batSentAt, batApprovedAt, formeOrderedAt, formeDeliveredAt)
- New FormeStatus type for die-cutting tool tracking
- Blocking logic at element level instead of job level

## New Types to Document

### Status Types (on Element)

```typescript
// PaperStatus (replaces job-level PaperPurchaseStatus)
type PaperStatus = 'none' | 'in_stock' | 'to_order' | 'ordered' | 'delivered';

// BatStatus (replaces job-level ProofApproval)
type BatStatus = 'none' | 'waiting_files' | 'files_received' | 'bat_sent' | 'bat_approved';

// PlateStatus (replaces job-level PlatesStatus)
type PlateStatus = 'none' | 'to_make' | 'ready';

// FormeStatus (NEW - for die-cutting tools)
type FormeStatus = 'none' | 'in_stock' | 'to_order' | 'ordered' | 'delivered';
```

### Date Tracking Fields (on Element)

```typescript
interface Element {
  // ... existing fields ...

  // Prerequisite status
  paperStatus: PaperStatus;
  batStatus: BatStatus;
  plateStatus: PlateStatus;
  formeStatus: FormeStatus;

  // Date tracking
  paperOrderedAt?: string;      // ISO timestamp
  paperDeliveredAt?: string;    // ISO timestamp
  filesReceivedAt?: string;     // ISO timestamp
  batSentAt?: string;           // ISO timestamp
  batApprovedAt?: string;       // ISO timestamp
  formeOrderedAt?: string;      // ISO timestamp
  formeDeliveredAt?: string;    // ISO timestamp
}
```

### Blocking Logic

An element is **blocked** if ANY prerequisite is not ready:
- Paper: ready states are `'none'`, `'in_stock'`, `'delivered'`
- BAT: ready states are `'none'`, `'bat_approved'`
- Plates: ready states are `'none'`, `'ready'`
- Forme: ready states are `'none'`, `'in_stock'`, `'delivered'`
