## Overview

This release establishes the **foundational types and backend entities** for the outsourcing refactor. It adds new fields to OutsourcedProvider and OutsourcedTask entities to support the new model where outsourced tasks become precedence constraints rather than scheduled tiles.

The key changes:
- **OutsourcedProvider** gains `transitDays` field for transport duration
- **OutsourcedTask** gains `manualDeparture` and `manualReturn` fields for user overrides
- Backward compatibility maintained for existing data

## Technical Notes

### New Field Specifications

**OutsourcedProvider.transitDays:**
- Type: `integer`
- Default: `1`
- Constraint: Must be positive (>= 1)
- Represents business days for transport (same value for outbound and return)

**OutsourcedTask.manualDeparture:**
- Type: `datetime|null`
- Default: `null`
- When set, overrides automatic departure calculation
- Format: ISO 8601 datetime string

**OutsourcedTask.manualReturn:**
- Type: `datetime|null`
- Default: `null`
- When set, overrides automatic return calculation
- Format: ISO 8601 datetime string

### Backward Compatibility

The existing `duration` structure in OutsourcedTask remains unchanged in this release:
```typescript
duration: {
  openDays: number;      // Will be renamed to workDays in v0.5.12
  latestDepartureTime: string;
  receptionTime: string;
}
```

The new fields are **additive** - no breaking changes to existing data or API contracts.
