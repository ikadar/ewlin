## Overview

Implement field linking between elements and a dual-mode (Form/JSON) template editor. Link propagation allows fields in subsequent elements to automatically inherit values from previous elements, reducing manual data entry. The JSON editor provides power users with direct template manipulation capabilities.

## Scope

### In Scope

- **Link Toggle per Field** — Toggle link state for: format, papier, imposition, impression, surfacage
- **Value Inheritance** — When link activated, copy value from previous element
- **Auto-Propagation** — When source value changes, automatically update linked fields
- **Visual Indicators** — Blue text/background for linked fields, link/unlink icons
- **Dual-Mode Template Editor** — Form tab (existing) + JSON tab with CodeMirror
- **JSON Editor** — CodeMirror 6 integration with syntax highlighting
- **Bidirectional Sync** — Changes in Form reflect in JSON and vice versa
- **Contextual Autocomplete in JSON** (Deferred to v0.4.36)

### Out of Scope

- Contextual autocomplete in JSON editor (v0.4.36)
- Template versioning/history
- Backend API integration (templates stored in localStorage)
- Link propagation for non-linkable fields (quantite, precedences, sequence, etc.)

## Technical Notes

### Linkable Fields (from §4 of spec)

Only these fields can be linked to the previous element:
- `format`
- `papier`
- `imposition`
- `impression`
- `surfacage`

### Link Behavior

1. **First element cannot have links** — No previous element to link to
2. **When link is activated**: Copy value from previous element immediately
3. **When source value changes**: Automatically propagate to all linked downstream elements
4. **Visual**: Blue text (`text-blue-400`) + blue background (`bg-blue-900/30`) when linked

### Link Data Model

```typescript
// Extension to JcfTemplateElement in @flux/types
export interface JcfTemplateElement {

}
```
