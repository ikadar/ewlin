# v0.6.6 - Aggressive Token Reduction

**Status:** In Progress
**Branch:** `feature/mcp-llm-scheduler`
**Date:** 2026-03-29

## Summary

Aggressively reduces token consumption across all dimensions: shorter system prompt, compact tool definitions, smaller max_tokens, tighter truncation limits, and leaner snapshot summaries.

## Changes

| What | Before | After | Reduction |
|------|--------|-------|-----------|
| System prompt | 3,690 chars | 826 chars | **78%** |
| Tool definitions | 11,403 chars | 7,459 chars | **35%** |
| Static tokens/call | ~3,773 | ~2,071 | **45%** |
| max_tokens (output cap) | 4,096 | 1,024 | **75%** |
| Conversation history cap | 20 messages | 12 messages | **40%** |
| Tool result truncation | 4,000 chars | 2,000 chars | **50%** |

### System Prompt
Reduced from 50 lines of verbose rules to 10 lines of essential directives. Removed examples, verbose explanations, section headers. Kept only actionable rules.

### Tool Definitions
Replaced verbose `Anthropic.Tool` objects with a compact `tool()` helper. Descriptions cut to single sentences. Property descriptions reduced to 2-3 words.

### Snapshot Summarizer
Stations and jobs now returned as compact strings (`"G37(uuid)"`, `"REF-001:planned(uuid)"`) instead of full objects with multiple fields.

### max_tokens
Reduced from 4,096 to 1,024. The LLM should give short answers and tool calls, not essays. This forces conciseness.
