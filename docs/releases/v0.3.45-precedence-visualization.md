## Overview

Add visual feedback during drag operations showing precedence constraints. Two horizontal lines will appear in the hovered station column: a purple line for the earliest possible start (predecessor end + optional dry time) and an orange line for the latest possible start (successor start - task duration).

**Implements:** REQ-10 from [new-requirements-02.md](../ux-ui/tmp/new-requirements-02.md)

## Scope

### In Scope

**REQ-10: Precedence Constraint Visualization**

| Element | Description |
|---------|-------------|
| **Purple line** | Earliest possible start (predecessor.scheduledEnd + dry time if printing) |
| **Orange line** | Latest possible start (successor.scheduledStart - currentTask.duration) |
| **Visibility** | Only during drag, only in hovered column |
| **Calculation** | Reuse `getEffectivePredecessorEnd` from @flux/schedule-validator |

**Visibility rules:**
- Lines appear only during drag operations
- Lines appear only in the station column being hovered
- Purple line: only if predecessor task is scheduled
- Orange line: only if successor task is scheduled
- No lines if no constraints exist

### Out of Scope

- Click interactions with the lines
- Displaying constraint information in tooltips
- Lines in non-hovered columns
- Lines outside of drag operations

## Technical Notes

### Calculation Logic

**Earliest possible (purple line):**
```typescript
import { getEffectivePredecessorEnd } from '@flux/schedule-validator';

// If predecessor is scheduled
if (predecessorAssignment) {
  const earliestStart = getEffectivePredecessorEnd(snapshot, predecessorAssignment);
  // Convert to Y position using timeToYPosition
}
```

**Latest possible (orange line):**
```typescript
// If successor is scheduled
if (successorAssignment) {
  const successorStart = new Date(successorAssignment.scheduledStart);
  const taskDurationMs = currentTask.estimatedDuration * 60 * 1000;
  const latestStart = new Date(successorStart.getTime() - taskDurationMs);
  // Convert to Y position using timeToYPosition
}
```

### Visual Styling

Based on `PlacementIndicator` component:
```tsx
// Purple line (earliest)
<div
  className="absolute left-0 right-0 h-1 bg-purple-500 z-30 pointer-events-none"
  style={{
    top: `${earliestY}px`,
    boxShadow: '0 0 12px rgba(168, 85, 247, 0.8)', // purple-500 glow
  }}
/>

// Orange line (latest)
<div
  className="absolute left-0 right-0 h-1 bg-orange-500 z-30 pointer-events-none"
  style={{
    top: `${latestY}px`,
    boxShadow: '0 0 12px rgba(249, 115, 22, 0.8)', // orange-500 glow
  }}
/>
```

### Data Flow

1. `App.tsx` calculates constraint positions during drag (onDrag monitor)
2. Pass `earliestStartY` and `latestStartY` to `SchedulingGrid`
3. `SchedulingGrid` passes to target `StationColumn`
4. `StationColumn` renders `PrecedenceLines` component

## v0.3.45 – Precedence Constraint Visualization

**Release Date:** 2026-01-06

### Summary

Visual feedback during drag operations showing task precedence constraints. Purple and orange lines indicate the valid placement range based on predecessor and successor tasks.

### What's New

- REQ-10: Precedence constraint lines during drag
  - Purple line: earliest possible start (predecessor end + dry time)
  - Orange line: latest possible start (successor start - task duration)
  - Lines visible only in hovered station column
  - Dry time (4h) automatically included for printing tasks

### Visual Changes

When dragging a task, colored lines appear in the hovered station column:
- **Purple line**: Shows where the task CAN start (after predecessor completes)
- **Orange line**: Shows where the task MUST start by (before successor)
- Lines have a glow effect for visibility
