## Overview

Introduces the `Element` entity to the type system, enabling jobs to have multiple production elements (e.g., cover, interior, finishing) with inter-element precedence rules. This is the foundational data model change for the Job Elements feature.

## Technical Notes

The `Element` entity represents a segment of the production process:
- **Physical parts**: cover (couv), interior (int)
- **Process steps**: finishing (fin), assembly

Elements have:
- `suffix`: short identifier (e.g., "couv", "int", "fin", "ELT")
- `prerequisiteElementIds`: elements that must complete before this one starts
- `taskIds`: tasks within this element (strict sequence)

Inter-element precedence only affects the **first task** of the dependent element.

### What's New

### Breaking Changes

- `Job.taskIds` replaced by `Job.elementIds`

### Migration Guide

Tasks should be accessed via their parent Element. Use `element.taskIds` to get tasks for an element.
