# v0.6.1 - AI Chat Model Selector

**Status:** In Progress
**Branch:** `feature/mcp-llm-scheduler`
**Date:** 2026-03-28

## Summary

Adds a model selector to the AI Scheduler chat panel, allowing operators to switch between Claude Sonnet 4 (higher capability) and Claude Haiku 4.5 (faster, lower cost) per message.

## Changes

### @flux/types
- Added `ChatModelId` type: `'sonnet' | 'haiku'`
- Added optional `model` field to `ChatRequest`

### MCP Server
- `client.ts`: Added `resolveModel()` function mapping `'sonnet'` / `'haiku'` to API model IDs
- `chat.ts`: Chat request schema accepts optional `model` parameter (validated via Zod enum)
- Model is passed through to `processMessage()` and used in `anthropic.messages.create()`
- Invalid model values are rejected with 400

### React Frontend
- `ChatPanel.tsx`: Added tabbed model selector (Sonnet / Haiku) in panel header
- `chatApi.ts`: `sendMessage` mutation accepts optional `model` parameter
- Default model: Sonnet
- Model selection is per-panel (persists across messages in the same session)

## Model Mapping

| UI Label | `ChatModelId` | API Model ID |
|----------|---------------|--------------|
| Sonnet | `sonnet` | `claude-sonnet-4-20250514` |
| Haiku | `haiku` | `claude-haiku-4-5-20251001` |

## Testing

### MCP Server (6 new tests)
- `resolveModel('sonnet')` returns correct API model ID
- `resolveModel('haiku')` returns correct API model ID
- `resolveModel(undefined)` defaults to Sonnet
- `resolveModel('gpt-4')` defaults to Sonnet
- Valid model parameter accepted in chat request
- Invalid model parameter rejected with 400

- MCP Server: TypeScript clean
- Frontend: TypeScript clean
