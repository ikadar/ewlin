## Overview

Level 3 Submit Validation for the Job Creation Form. When the user clicks Save, perform strict full-form validation:
- Convert all required fields (amber indicators) to errors
- Apply strict DSL format validation (no leniency for incomplete input)
- Block form submission until all errors are resolved
- Display error summary and scroll to first error

This completes the 3-level validation system:
- **Level 1** (v0.4.23): Live format validation during typing
- **Level 2** (v0.4.24): Required field indicators (amber dots)
- **Level 3** (v0.4.30): Strict submit validation with error blocking

## Scope

### In Scope

- `hasAttemptedSubmit` state flag for triggering Level 3 errors
- `validateForSubmit()` function that:
  - Applies strict DSL validation (same as Level 1 but no leniency)
  - Converts required fields to submit errors
- Error messages in French with HTML formatting for tooltips
- Integration with Save button via ref callback pattern
- Form blocking (Save returns false if errors exist)

### Out of Scope

- Actual API save (handled by future release)
- Error summary panel (errors shown per-cell, same as Level 1)
- Scroll to first error (can be added later)
- Template save (templates skip validation)
