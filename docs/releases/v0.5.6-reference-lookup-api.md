# Release v0.5.6 – Reference Lookup via API

> **Status:** Released
>
> **Milestone:** M5 (Phase 5B: JCF API Integration)
>
> **Target Date:** 2026-02-03
>
> **Git Tag:** `v0.5.6`

---

## Overview

This release adds reference lookup functionality to the JCF form. When the user enters a job ID that matches an existing job in the system, the client field is automatically populated with the associated client name. This improves data entry efficiency and reduces errors.

**Key deliverable:** Auto-fill client field when entering an existing job reference.

---

## Prerequisites

- [x] `v0.5.0` API Client Configuration (hybrid base query)
- [x] `v0.5.5` Client Autocomplete via API (RTK Query pattern)

---

## Scope

### In Scope

- Add `lookupByReference` RTK Query endpoint in `scheduleApi.ts`
- Add mock handler for `GET /jobs/lookup-by-reference?ref={ref}` in `mockBaseQuery.ts`
- Update `JcfJobHeader` to trigger lookup on job ID blur
- Auto-fill client field when existing job is found
- Handle "not found" gracefully (no error, just no auto-fill)

### Out of Scope

- Full job data auto-fill (only client field for now)
- Reference validation (checking format)
- Error handling for API failures (v0.5.7)

---

## Technical Notes

### API Response Type

```typescript
// In @flux/types
export interface ReferenceLookupResponse {
  found: boolean;
  clientName?: string;
}
```

### Lookup Trigger

- Lookup is triggered on **blur** of the job ID field
- Only triggers if job ID is non-empty and client field is empty
- Uses debounced value to avoid rapid API calls during editing

### Mock Data

The mock handler returns client name for existing jobs in the mock snapshot.
