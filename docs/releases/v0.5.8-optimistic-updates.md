## Overview

This release implements optimistic updates for all assignment operations, providing immediate UI feedback before API responses arrive. When a user performs an assignment operation (create, move, or recall), the UI updates instantly. If the API call fails, the change automatically rolls back to the previous state.

**Key deliverable:** Instant UI feedback for all scheduling operations with automatic rollback on error.

## Technical Notes

### Existing Pattern (toggleCompletion)

The `toggleCompletion` mutation already uses optimistic updates:

```typescript
toggleCompletion: build.mutation<CompletionResponse, string>({
  query: (taskId) => ({ url: `/tasks/${taskId}/completion`, method: 'PUT' }),
  invalidatesTags: ['Snapshot'],
  async onQueryStarted(taskId, { dispatch, queryFulfilled }) {
    const patchResult = dispatch(
      scheduleApi.util.updateQueryData('getSnapshot', undefined, (draft) => {
        const assignment = draft.assignments.find((a) => a.taskId === taskId);
        if (assignment) {
          assignment.isCompleted = !assignment.isCompleted;
          assignment.completedAt = assignment.isCompleted ? new Date().toISOString() : null;
          assignment.updatedAt = new Date().toISOString();
        }
      })
    );
    try {
      await queryFulfilled;
    } catch {
      patchResult.undo();
    }
  },
}),
```

### Pattern to Apply

The same pattern will be applied to:

1. **assignTask** - Add new assignment to `draft.assignments`
2. **rescheduleTask** - Update existing assignment in `draft.assignments`
3. **unassignTask** - Remove assignment from `draft.assignments`

### Key Considerations

- **scheduledEnd calculation**: Must calculate duration from task data in snapshot
- **ID generation**: Temporary ID for optimistic assignment (replaced by API response)
- **Push-down**: NOT handled optimistically (complex logic, server authoritative)
- **Rollback**: Automatic via `patchResult.undo()` on error
