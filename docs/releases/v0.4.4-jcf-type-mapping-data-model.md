## Overview

Maps the reference JCF Element data model (12 string fields) to the existing `@flux/types` model. This is the foundational type mapping release that all subsequent JCF releases (v0.4.5–v0.4.28) depend on.

The reference JCF `Element` has 12 string fields for production metadata (format, papier, pagination, imposition, impression, surfacage, quantite, qteFeuilles, autres, commentaires, sequence, precedences). These need to be represented as an `ElementSpec` value object on the existing `@flux/types` `Element` interface, plus associated reference data types for autocomplete support.

## Technical Notes

### Architectural Decision: ElementSpec as Value Object

The reference JCF `Element` stores all production metadata as 12 flat string fields. For Ordo, we add an `ElementSpec` value object to the existing `Element` interface:

```typescript
// New: Production metadata value object
interface ElementSpec {
  format?: string;        // Product format (e.g., "A4", "210x297", "A3/A6")
  papier?: string;        // Paper DSL (e.g., "Couché mat:135")
  pagination?: number;    // Page count (2 or multiple of 4)
  imposition?: string;    // Sheet imposition (e.g., "50x70(8)")
  impression?: string;    // Printing spec (e.g., "Q/Q", "Q+V/")
  surfacage?: string;     // Finishing/coating (e.g., "mat/mat", "brillant/")
  quantite?: number;      // Element-level quantity multiplier
  qteFeuilles?: number;   // Calculated: ceil(jobQty × elementQty / poses)
  autres?: string;        // Free-text other notes
  commentaires?: string;  // Free-text comments
}

// Extended Element interface
interface Element {
  // ... existing fields (id, jobId, suffix, label, etc.)
  spec?: ElementSpec;     // Production metadata (JCF fields)
}
```

**Why `ElementSpec` value object vs flat fields:**
1. **Separation of concerns**: Scheduling fields (id, jobId, taskIds) stay clean
2. **Optional**: Elements without production metadata (scheduling-only) remain valid
3. **Serializable**: Value object maps cleanly to JSON storage
4. **Grouping**: All 10 production fields are logically grouped
5. **Extensible**: New production fields can be added without touching Element interface

### Field Mapping: Reference JCF → @flux/types

| JCF Element Field | @flux/types Field | Type | Notes |
|-------------------|-------------------|------|-------|
| `name` | `Element.suffix` | `string` | Already exists |
| `precedences` | `Element.prerequisiteElementIds` | `string[]` | Already exists (by ID, not name) |
| `sequence` | `Element.taskIds` → `Task[]` | — | Tasks are separate entities, not stored on Element |
| `format` | `ElementSpec.format` | `string?` | DSL: "A4", "210x297", "A3/A6" |
| `papier` | `ElementSpec.papier` | `string?` | DSL: "Couché mat:135" |
| `pagination` | `ElementSpec.pagination` | `number?` | 2 or multiple of 4 |
| `imposition` | `ElementSpec.imposition` | `string?` | DSL: "50x70(8)" |
| `impression` | `ElementSpec.impression` | `string?` | DSL: "Q/Q", "Q+V/" |
| `surfacage` | `ElementSpec.surfacage` | `string?` | DSL: "mat/mat", "brillant/" |
| `quantite` | `ElementSpec.quantite` | `number?` | Element-level multiplier |
| `qteFeuilles` | `ElementSpec.qteFeuilles` | `number?` | Calculated or manual |
| `autres` | `ElementSpec.autres` | `string?` | Free text |
| `commentaires` | `ElementSpec.commentaires` | `string?` | Free text |

### Reference Data Types (New)

These types represent the reference/lookup data needed by autocomplete fields:

```typescript
// Paper reference data
interface PaperType {
  id: string;
  type: string;           // e.g., "Couché mat"
  grammages: number[];    // e.g., [70, 80, 90, ..., 400]
}

// Sheet format (for imposition)
interface FeuilleFormat {
  format: string;         // e.g., "50x70"
  poses: number[];        // e.g., [1, 2, 4, 8, 16, 32, 64, 128]
}

// Product format
interface ProductFormat {
  id: string;
  name: string;           // e.g., "A4", "A4f", "SRA3"
  width: number;          // mm
  height: number;         // mm
}

// Impression preset
interface ImpressionPreset {
  id: string;
  value: string;          // e.g., "Q/Q"
  description: string;    // e.g., "Quadri R/V"
}

// Surfacage preset
interface SurfacagePreset {
  id: string;
  value: string;          // e.g., "mat/mat"
  description: string;    // e.g., "Pelli mat R/V"
}

// Poste category
type PosteCategory =
  | 'Presse offset'
  | 'Presse numérique'
  | 'Massicot'
  | 'Typo'
  | 'Plieuse'
  | 'Encarteuse-piqueuse'
  | 'Assembleuse-piqueuse'
  | 'Conditionnement'

// Poste preset
interface PostePreset {
  name: string;           // e.g., "G37", "Komori"
  category: PosteCategory;
}

// Sous-traitant preset
interface SoustraitantPreset {
  name: string;           // e.g., "MCA", "F37"
}

// Template category
interface TemplateCategory {
  id: string;
  name: string;           // e.g., "Brochure", "Feuillet"
}
```
