## Overview

Extends the existing `JcfAutocomplete` component (built in v0.4.8) with table cell navigation delegation. Adds `onTabOut` and `onArrowNav` callback props so autocomplete fields inside the Elements Table (v0.4.13+) can participate in the grid's Tab/Alt+Arrow navigation (v0.4.10).

The base autocomplete features (dropdown, filtering, text highlighting, keyboard nav, lazy loading, click-outside) were already implemented in v0.4.8. This release adds the table-integration layer.

## Scope

### In Scope

- **`onTabOut` prop**: When Tab/Shift+Tab is pressed (dropdown closed or just closed), delegate to parent table's navigation handler
- **`onArrowNav` prop**: When Alt+Arrow is pressed, delegate to parent table's navigation handler
- **Keyboard handler update**: Alt+Arrow always delegates (closes dropdown if open first); Tab closes dropdown then delegates
- **Unit tests** for delegation callbacks
- **E2E test** verifying autocomplete delegates Tab/Alt+Arrow to table navigation

### Out of Scope

- Session learning (v0.4.12)
- Integration of autocomplete into specific table cells (v0.4.13+)
- New autocomplete features (DSL parsing, multi-step selection)

## Technical Notes

### Delegation Pattern

When an autocomplete field is inside the Elements Table, it receives `onTabOut` and `onArrowNav` callbacks from the table. The autocomplete delegates keyboard events that should trigger table navigation:

```
User presses Tab in autocomplete
  → Autocomplete closes dropdown (if open)
  → Calls onTabOut(event, 'forward')
  → Table's handleCellKeyDown processes the Tab
  → Focus moves to next cell in the grid
```

### Keyboard Handler Changes

| Key | Dropdown Open | Dropdown Closed |
|-----|--------------|----------------|
| **Tab** | Close dropdown → delegate to `onTabOut` | Delegate to `onTabOut` (if provided, else native Tab) |
| **Shift+Tab** | Close dropdown → delegate to `onTabOut('backward')` | Delegate to `onTabOut('backward')` |
| **Alt+Arrow** | Close dropdown → delegate to `onArrowNav` | Delegate to `onArrowNav` |
| **↑↓ (no Alt)** | Navigate dropdown suggestions | Open dropdown |
| **Enter** | Select suggestion | No change |
| **Escape** | Close dropdown | No change |

### Props Addition

```typescript
interface JcfAutocompleteProps {
  // ... existing props ...
  onTabOut?: (e: React.KeyboardEvent, direction: 'forward' | 'backward') => void;
  onArrowNav?: (e: React.KeyboardEvent, direction: 'up' | 'down' | 'left' | 'right') => void;
}
```

When `onTabOut` is not provided, Tab behaves as before (native browser Tab). This preserves backward compatibility with Job Header usage (v0.4.8).
