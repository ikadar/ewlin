## Overview

Replaces the plain text inputs for the Quantité and Pagination fields in the Elements Table with specialized numeric input components. Quantité filters to digits-only, defaults to "1" on blur, and supports Arrow Up/Down increment/decrement. Pagination filters to digits-only with inline validation — valid values are 2 (feuillet) or multiples of 4 (cahier: 4, 8, 12, 16, ...). Invalid pagination values show a red border as visual error feedback.

Both components support table navigation delegation (Tab/Shift+Tab, Alt+Arrow).

## Technical Notes

### Quantité Input Behavior

- **Input filter**: Digits only (`/\D/g` → strip), strip leading zeros
- **Blur default**: If empty or "0", default to "1"
- **Arrow Up**: Increment current value by 1
- **Arrow Down**: Decrement current value by 1 (minimum 1)
- **Placeholder**: "1"
- **Alignment**: `text-right`
- **Input mode**: `inputMode="numeric"` (mobile keyboard hint)

### Pagination Input Behavior

- **Input filter**: Digits only (`/\D/g` → strip), strip leading zeros
- **No blur default**: Empty is allowed (field may be optional)
- **Validation visual**: Red border (`border-red-500`) when value is non-empty and invalid
- **Alignment**: `text-right`
- **Input mode**: `inputMode="numeric"` (mobile keyboard hint)

### Pagination Validation (§1.6)

```typescript
function isValidPagination(value: string): boolean {
  if (value === '') return true; // empty is valid (optional field)
  const num = parseInt(value, 10);
  if (isNaN(num) || num < 1) return false;
  return num === 2 || (num >= 4 && num % 4 === 0);
}
```

Valid: `2`, `4`, `8`, `12`, `16`, `20`, ...
Invalid: `1`, `3`, `5`, `6`, `7`, `9`, `10`, `11`, ...

### Component Architecture

```
JcfElementsTable
  ├── JcfQuantiteInput (row.key === 'quantite')
  │     ├── digits-only filter
  │     ├── blur → default "1"
  │     ├── ArrowUp/Down increment/decrement
  │     └── onTabOut/onArrowNav delegation
  │
  └── JcfPaginationInput (row.key === 'pagination')
        ├── digits-only filter
        ├── isValidPagination check
        ├── red border on invalid
        └── onTabOut/onArrowNav delegation
```

### JcfElementsTable Integration

- Detect `row.key === 'quantite'` → render JcfQuantiteInput
- Detect `row.key === 'pagination'` → render JcfPaginationInput
- Both with:
  - `value={element[row.key]}`
  - `onChange={(v) => handleCellChange(elementIndex, row.key, v)}`
  - `onTabOut` / `onArrowNav` → delegate to cell navigation
  - `id={getCellId(elementIndex, rowIndex)}`
