## Overview

Add visual feedback on scheduler tiles to indicate when an element is blocked due to missing prerequisites. Blocked tiles show a dashed left border, and hovering for 2 seconds displays a tooltip with prerequisite status details.

## Design

### Blocking Logic

An element is "blocked" if any prerequisite is not in a ready state:

```typescript
function isElementBlocked(element: Element): boolean {
  const paperReady = ['none', 'in_stock', 'delivered'].includes(element.paperStatus);
  const batReady = ['none', 'bat_approved'].includes(element.batStatus);
  const platesReady = ['none', 'ready'].includes(element.plateStatus);

  return !paperReady || !batReady || !platesReady;
}
```

### Tile Visual Changes

| State | Left Border Style |
|-------|-------------------|
| Ready (not blocked) | Solid 4px job color |
| Blocked | Dashed 4px job color |

CSS for dashed border:
```css
border-left: 4px dashed var(--job-color);
```

### Tooltip Design

Tooltip appears after **2 seconds** of hover on a **blocked** tile only.

**Content format:**
```
⚠ Papier: Commandé
⚠ BAT: Attente fichiers
✓ Plaques: Prêtes
```

- ⚠ (warning) for blocking items (not ready)
- ✓ (check) for ready items
- Only show relevant prerequisites (hide Plates if `none`)

### Tooltip Styling

- Dark background (`bg-slate-800`)
- White text
- Rounded corners
- Shadow
- Positioned above tile, centered
- Arrow pointing down
