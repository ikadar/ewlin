# Release v0.5.10 – Remove Provider Columns

> **Status:** 🔴 Not Started
>
> **Milestone:** M5 (Phase 5D: Outsourcing Refactor)
>
> **Target Date:** 2026-02-04
>
> **Git Tag:** `v0.5.10`

---

## Overview

This release removes the provider columns from the scheduling grid as part of the outsourcing refactor. Outsourced tasks will no longer appear as visible tiles in the grid - they become invisible precedence constraints (like drying time).

This is a **focused removal release**, not a feature addition. The goal is to simplify the grid by eliminating the outsourcing visualization that will be replaced by a new model in subsequent releases.

## v0.5.10 – Remove Provider Columns

**Release Date:** 2026-02-04

### Summary

Remove provider columns from the scheduling grid as part of the outsourcing
refactor. Outsourced tasks will be managed through the Job Details Panel
instead of appearing as tiles in the grid.

### What's Changed

- Removed ProviderColumn and ProviderHeader components
- Removed provider column rendering from SchedulingGrid
- Removed isOutsourced and subcolumnLayout props from Tile component
- Simplified grid layout to show only station columns

### Breaking Changes

None for end users. Provider columns no longer appear in the grid.

### Migration Guide

No migration required. Outsourced tasks are now managed only through
the Job Details Panel task list.

### Notes

This is part of Phase 5D (Outsourcing Refactor). Subsequent releases will add:
- v0.5.11: Editable mini-form for outsourcing parameters
- v0.5.12: New precedence calculation with transit days
- v0.5.13: Drag visualization showing outsourcing constraints
