## Overview

A v0.5.18 (API Integration) csak az in-memory RTK Query cache-t frissítette prerequisite változtatáskor — az oldalfrissítés elveszítette a változtatásokat. Ez a release hozzáadja a perzisztálást: a listbox-ban végzett módosítás egy `PATCH /api/v1/flux/elements/{id}` hívást indít, és az adatbázisba ment.

RTK Query `onQueryStarted` optimista update pattern: a cache azonnal frissül, és hibás API hívás esetén automatikusan visszaáll (undo).

### PHP API (`services/php-api`)

- **`PATCH /api/v1/flux/elements/{id}`** — egyetlen prerequisite mező frissítése
  - Request body: `{ "column": "bat"|"papier"|"formes"|"plaques", "value": "<enum string>" }`
  - Response: 200 + `FluxElementResponse` JSON, vagy 404/422 hiba esetén
  - Enum mapping: `BatStatus::tryFrom($value)`, `PaperStatus::tryFrom()`, stb.

  - Element GUID alapján keresi az entity-t (`ElementRepository`)

- **`fluxApi.ts`** — `updateElementPrerequisite` RTK Query mutation
  - `PATCH /flux/elements/{elementId}` a mock/real base query-n keresztül
  - `onQueryStarted`: optimistic cache update + undo on error
- **`mockBaseQuery.ts`** — `PATCH /flux/elements/{id}` mock handler
  - In-memory `FLUX_STATIC_JOBS` frissítése (tesztelhetőség)
- **`FluxPage.tsx`** — `handleUpdatePrerequisite` mutation-t hív a dispatch helyett

## Technical Notes

### RTK Query optimistic update pattern

```ts
updateElementPrerequisite: builder.mutation<void, UpdatePrerequisiteArg>({
  query: ({ elementId, column, value }) => ({
    url: `/flux/elements/${elementId}`,
    method: 'PATCH',
    body: { column, value },
  }),
  async onQueryStarted({ elementId, column, value }, { dispatch, queryFulfilled }) {
    const patchResult = dispatch(
      fluxApi.util.updateQueryData('getFluxJobs', undefined, (draft) => {
        for (const job of draft) {
          const el = job.elements.find(e => e.id === elementId);
          if (el) { el[column] = value; break; }
        }
      }),
    );
    try {
      await queryFulfilled;
    } catch {
      patchResult.undo();
    }
  },
}),
```

### Mock mode

A `mockBaseQuery.ts` `PATCH /flux/elements/{id}` kérésekre `{ data: {} }` választ ad — az optimistic update már megtörtént az `onQueryStarted`-ben, a mock response nincs felhasználva.

### Element ID

A `FluxElement.id` mező: mock módban `'00042-e1'` stílusú, valós API módban PHP GUID. A frontend mindkét esetben ugyanúgy kezeli. A PATCH URL-ben az `elementId`-t az `element.id` mezőből vesszük.
