# v0.6.4 - Token Optimization

**Status:** In Progress
**Branch:** `feature/mcp-llm-scheduler`
**Date:** 2026-03-28

## Summary

Reduces Claude API token consumption by ~60% through four complementary optimizations: prompt caching, conversation truncation, tool result truncation, and defaulting to Haiku.

## Problem

Each chat message was sending the full system prompt (~800 tokens) + 27 tool definitions (~3K tokens) + unbounded conversation history to Claude, uncached. A single user interaction with 3-5 agentic loop rounds could consume 50K+ tokens. Large tool results (schedule snapshots with 50+ jobs) bloated context further.

## Optimizations

### 1. Prompt Caching (biggest win: ~90% savings on static content)

The system prompt and tool definitions are identical across every API call. By adding `cache_control: { type: 'ephemeral' }` to the system block and the last tool definition, Anthropic caches these at 90% discount after the first call.

**Before:** ~3,800 input tokens per call for system + tools
**After:** ~380 tokens per call (after first cache fill)

```typescript
system: [{
  type: 'text',
  text: systemPrompt,
  cache_control: { type: 'ephemeral' },
}],
tools: [...toolDefinitions, { ...lastTool, cache_control: { type: 'ephemeral' } }],
```

### 2. Conversation Truncation (caps runaway growth)

Conversations are now limited to 20 messages. When exceeded, the first message (initial context) is preserved plus the most recent 19. This prevents long multi-step conversations from growing indefinitely.

**Constant:** `MAX_CONVERSATION_MESSAGES = 20`

### 3. Tool Result Truncation (caps large payloads)

Tool results (especially `get_schedule_summary` with many jobs) are capped at 4,000 characters. Truncated results include a notice with the number of omitted characters.

**Constant:** `MAX_TOOL_RESULT_CHARS = 4000`

### 4. Default Model: Haiku

The default model is now Haiku 4.5 instead of Sonnet 4. Most scheduling commands are straightforward pattern matching (lookup + call tool) that Haiku handles well at ~10x lower cost. Users can still select Sonnet in the UI for complex reasoning.

**Cost comparison (approximate):**
| Model | Input | Output |
|-------|-------|--------|
| Haiku 4.5 | $0.80/MTok | $4/MTok |
| Sonnet 4 | $3/MTok | $15/MTok |

## Estimated Token Savings

| Optimization | Saving |
|-------------|--------|
| Prompt caching | ~3,400 tokens/call (90% of 3,800) |
| Conversation truncation | Variable — prevents 100K+ token conversations |
| Tool result truncation | Up to 10K+ per large result |
| Haiku default | ~75% cost reduction per token |

**Combined estimated reduction:** ~60% cost reduction for typical usage patterns.
