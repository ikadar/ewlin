## Overview

Adds workflow-guided suggestion ordering to the Sequence autocomplete. When an element has a `sequenceWorkflow` array (defined in template mode), the poste suggestions in the dropdown are reordered to prioritize postes matching the expected production step. Matching postes are marked with a star prefix in their category badge.

This is the third and final Sequence release (v0.4.20 Poste mode, v0.4.21 ST mode, v0.4.22 Workflow suggestions).

## Scope

### In Scope

- **`sequenceWorkflow` prop**: Optional `string[]` prop on JcfSequenceAutocomplete
- **Workflow step detection**: Count completed lines (with `)`) to determine current step index
- **Expected category lookup**: `sequenceWorkflow[stepIndex]` with comma-separated multi-category support
- **Priority sorting**: Postes matching expected category appear first in suggestions
- **Star marker**: Matching postes show `★ category` in description badge instead of plain category
- **Non-matching postes still shown**: All postes remain visible, just reordered
- **DSL utility functions**: `getWorkflowStepIndex()`, `getExpectedCategories()`
- **Unit tests** for DSL utilities and component priority behavior
- **E2E tests** for workflow-guided ordering

### Out of Scope

- Template mode workflow editor (`WorkflowSequenceAutocomplete` component -- separate feature)
- Persisting `sequenceWorkflow` to elements (managed by template system)
- Per-line error highlighting (v0.4.23)
- Filtering out already-used postes

---

## Technical Notes

- **Step index calculation**: Count lines with closing paren `)` before the cursor position. This represents the number of completed production steps.
- **Multi-category per step**: A workflow step like `"Presse offset, Presse numérique"` means either category matches for that step.
- **Sort stability**: Priority postes come first, non-priority keep their original order. Sort is stable (`a < b` when a matches and b doesn't).
- **Star prefix**: When a poste matches the expected category, its description changes from `"Presse offset"` to `"★ Presse offset"`.
- **Empty workflow**: When `sequenceWorkflow` is empty or undefined, no priority sorting occurs (default behavior from v0.4.20).
- **Beyond workflow length**: When `stepIndex >= workflow.length`, no priority sorting occurs (all steps completed).
- **Only affects poste step**: Workflow priority only applies to the `poste` suggestion step, not ST or duration steps.
