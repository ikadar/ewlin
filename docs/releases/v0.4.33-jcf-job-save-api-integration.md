## Overview

This release connects the existing JCF (Job Creation Form) frontend in `apps/web` with the PHP API backend. The JCF form is already fully implemented with validation - this release adds the actual API integration.

**Current State:**
- JCF form implemented in `apps/web/src/App.tsx` (lines 1626-1651)
- Save button triggers validation via `jcfSaveAttemptRef`
- Save is currently **simulated** with `setTimeout` (line 449-462)
- TODO comment: "Actual API save will be implemented in future release"

**Goal:** Replace simulated save with actual `POST /api/v1/jobs` API call.

## Scope

### In Scope

**Part 1: Naming Alignment (`suffix` → `name`) - ALL LAYERS**

This is a cross-cutting change affecting 3 packages:

| Package | Files | Changes |
|---------|-------|---------|
| `packages/types` | 1 file | `Element.suffix` → `Element.name` |
| `services/php-api` | 5 source + 9 test files | Entity, DTOs, events, migration |
| `apps/web` | 1 source + 12 test/mock files | Components, fixtures, tests |

**Part 2: Multi-Element Job Creation (services/php-api)**
- Extend `POST /api/v1/jobs` to accept `elements[]` array
- Create JcfElementInput DTO for element data
- Create multiple Elements per Job (not just default)
- Parse each element's `sequence` field into Tasks
- Handle element prerequisites by name lookup

**Part 3: Frontend API Integration (apps/web)**
- Replace simulated save in `handleJcfSave` with actual API call
- Transform JCF form state to API request format
- Success feedback (close modal, optionally show toast)
- Error handling (display validation errors)

### Out of Scope

- Template system (v0.4.34)
- Reference data endpoints (clients, papers, etc.)
- Navigation to job details after save

## Data Mapping

### JCF Form State → API Request

```
JCF State              → API Request Field
───────────────────────────────────────────
jcfJobId               → reference (auto-generated in frontend)
jcfClient              → client
jcfIntitule            → description
jcfDeadline            → workshopExitDate
jcfQuantity            → (stored in job metadata or ignored)
jcfTemplate            → (stored in job metadata or ignored)
jcfElements[]          → elements[]
```

### JcfElement → API Element Input

**No mapping needed** - frontend and backend now use the same naming:

```
JcfElement Field       → API Element Field
───────────────────────────────────────────
name                   → name (ELT, INT, COUV, etc.)
sequence               → sequence (parsed into Tasks via DSL)
precedences            → prerequisiteNames (resolved to IDs by backend)
format|papier|...      → label (composed by frontend)
quantite               → (metadata)
pagination             → (metadata)
imposition             → (metadata)
autres                 → (metadata)
qteFeuilles            → (metadata)
commentaires           → (metadata)
```

### Example API Request

```json
{
  "reference": "250131-001",
  "client": "Dupont",
  "description": "Brochure A4 16p",
  "workshopExitDate": "2026-02-15",
  "elements": [
    {
      "name": "INT",
      "label": "A4 | Couche 135g | Q/Q | mat/mat",
      "sequence": "[Komori] 20+40\n[Massicot] 15"
    },
    {
      "name": "COUV",
      "label": "A4 | Couche 250g | Q/Q | brillant/",
      "sequence": "[Komori] 15+30\n[Massicot] 10",
      "prerequisiteNames": ["INT"]
    }
  ]
}
```

## Release Notes Draft

```markdown
## v0.4.33 – JCF: Job Save & API Integration

**Release Date:** 2026-01-31

### Summary

Connects the Job Creation Form (JCF) frontend with the PHP API backend, enabling actual job creation with multiple elements. Also aligns backend naming conventions with frontend.

### What's New

**Naming Alignment (cross-cutting):**
- Element `suffix` renamed to `name` across all layers:
  - `@flux/types` package
  - PHP API backend
  - React frontend
- API responses now use `name` instead of `suffix`

**Backend:**
- POST /api/v1/jobs now accepts `elements[]` array for multi-element jobs
- Each element can have its own `sequence` field (Task DSL)
- Element prerequisites can reference other elements by name

**Frontend:**
- JCF form now saves to API (was previously simulated)
- Validation errors from API are displayed

### API Changes

**Breaking Change:** Element responses now use `name` instead of `suffix`:
```json
{
  "elements": [
    {
      "id": "...",
      "name": "INT",  // was: "suffix": "INT"
      "label": "..."
    }
  ]
}
```

POST /api/v1/jobs now accepts optional `elements` array:
```json
{
  "reference": "250131-001",
  "client": "Client Name",
  "description": "Job Description",
  "workshopExitDate": "2026-02-15",
  "elements": [
    {
      "name": "INT",
      "label": "Interior pages description",
      "sequence": "[Komori] 20+40",
      "prerequisiteNames": ["COUV"]
    }
  ]
}
```

If `elements` is not provided, a default element is created (backward compatible).
```
