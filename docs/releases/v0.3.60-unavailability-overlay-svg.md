## Overview

Replace the CSS gradient-based stripe pattern for the unavailability overlay with an SVG file. This change is motivated by **performance** improvements - SVG patterns are more efficiently rendered by browsers than CSS repeating gradients, especially when many overlays are visible on a multi-day grid.

## Technical Notes

### Current CSS Implementation

```css
.bg-stripes-dark {
  background-image: repeating-linear-gradient(
    45deg,
    transparent,
    transparent 10px,
    rgba(255, 255, 255, 0.03) 10px,
    rgba(255, 255, 255, 0.03) 20px
  );
}
```

### SVG Pattern Specifications

| Property | Value |
|----------|-------|
| Angle | 45° (diagonal) |
| Stripe color | `rgba(255, 255, 255, 0.03)` → `#ffffff` with 3% opacity |
| Stripe width | 10px |
| Gap width | 10px |
| Pattern repeat | 20px (diagonal distance) |
| Pattern size | ~28.28px × 28.28px (20px × √2 for 45° rotation) |

### SVG Implementation

```xml
<svg xmlns="http://www.w3.org/2000/svg" width="28.28" height="28.28">
  <defs>
    <pattern id="stripes" patternUnits="userSpaceOnUse" width="28.28" height="28.28">
      <rect width="28.28" height="28.28" fill="transparent"/>
      <line x1="0" y1="28.28" x2="28.28" y2="0" stroke="rgba(255,255,255,0.03)" stroke-width="14.14"/>
    </pattern>
  </defs>
  <rect width="100%" height="100%" fill="url(#stripes)"/>
</svg>
```

Alternative simpler approach with rotated rect:

```xml
<svg xmlns="http://www.w3.org/2000/svg" width="20" height="20" viewBox="0 0 20 20">
  <rect width="10" height="28.28" fill="rgba(255,255,255,0.03)" transform="rotate(-45 10 10)"/>
</svg>
```

### CSS Update

```css
.bg-stripes-dark {
  background-image: url('/stripes.svg');
  background-repeat: repeat;
}
```
