## Overview

Fix the inconsistency between visual snap position and validation during drag operations. Currently, the tile visually snaps to the 30-minute grid, but validation (which determines the border color) uses the raw cursor position. This causes confusing feedback where a green/red border doesn't match what the user sees.

**Implements:** REQ-01, REQ-02, REQ-03 from [refactored-new-requirements-02-en.md](../ux-ui/tmp/refactored-new-requirements-02-en.md#req-010203-drag-snapping-consistency)

## Technical Notes

### The Problem

```typescript
// Current (buggy) - App.tsx:230-246
function calculateScheduledStartFromPointer(...): string | null {
  const rect = stationElement.getBoundingClientRect();
  const absoluteY = calculateTileTopPosition(clientY, rect.top, grabOffsetY);

  const dropTime = yPositionToTime(absoluteY, START_HOUR, startDate);  // ❌ NO SNAP!
  return dropTime.toISOString();
}
```

### The Solution

```typescript
// Fixed
function calculateScheduledStartFromPointer(
  clientX: number,
  clientY: number,
  grabOffsetY: number,
  startDate?: Date,
  pixelsPerHour: number = PIXELS_PER_HOUR  // ← ADD PARAMETER
): string | null {
  const rect = stationElement.getBoundingClientRect();
  const absoluteY = calculateTileTopPosition(clientY, rect.top, grabOffsetY);

  const snappedY = snapToGrid(absoluteY, pixelsPerHour);  // ← ADD THIS
  const dropTime = yPositionToTime(snappedY, START_HOUR, startDate, pixelsPerHour);
  return dropTime.toISOString();
}
```

### Call Sites to Update

1. **Line ~581** (in `onDrag` handler):
   ```typescript
   const scheduledStart = calculateScheduledStartFromPointer(
     clientX, clientY, grabOffset.y, gridStartDate, pixelsPerHour
   );
   ```

2. **Line ~631** (in `onDrop` handler):
   ```typescript
   const calculatedScheduledStart = currentDragValidation.scheduledStart
     || calculateScheduledStartFromPointer(clientX, clientY, grabOffset.y, gridStartDate, pixelsPerHour);
   ```
