# v0.6.3 - Extend & Replan (Smart Duration Change)

**Status:** In Progress
**Branch:** `feature/mcp-llm-scheduler`
**Date:** 2026-03-28

## Summary

When an operator says "this task will take 3h instead of 1h", the system now properly handles the full cascading impact instead of just changing a number. The new `extend_task_and_replan` tool atomically performs the entire replanning workflow.

## Problem

Previously, `update_task_duration` only changed the task's run minutes and recalculated its own `scheduledEnd`. This left the schedule in an inconsistent state:
- Tasks on the same station that were scheduled immediately after now **overlap**
- Downstream tasks in the precedence chain (same element, dependent elements) become **unreachable** on time
- No indication of which jobs are now **late**

The LLM had to manually chain `update_task_duration` + `unassign_tasks` + `auto_place` — but it didn't know which tasks to unassign, making the result incomplete.

## Solution

### New PHP Service: `ReplanService`
**File:** `services/php-api/src/Service/ReplanService.php`

Orchestrates the full workflow:
1. **Update duration** — changes `runMinutes`, recalculates `scheduledEnd`
2. **Identify impacted tasks** using BFS traversal:
   - Same station: all tasks scheduled after the modified task
   - Intra-element precedence: successor tasks by `sequenceOrder`
   - Cross-element precedence: first tasks of dependent elements
   - Cascade: for each impacted task, find ITS same-station followers (domino effect)
3. **Filter**: skip completed tasks and tasks currently in progress
4. **Unassign** all impacted tasks
5. **Auto-place** via ASAP algorithm for each affected job
6. **Report** late jobs (where latest task end > job deadline)

### New PHP Endpoint
```
POST /api/v1/tasks/{taskId}/extend-and-replan
Body: { "runMinutes": 180 }
Response: {
  "taskId": "uuid",
  "previousRunMinutes": 60,
  "newRunMinutes": 180,
  "impactedTaskIds": ["uuid1", "uuid2", ...],
  "unassignedCount": 5,
  "replacedCount": 4,
  "lateJobs": [{ "jobId": "uuid", "reference": "4428" }]
}
```

### New MCP Tool: `extend_task_and_replan`
- Marked as **PREFERRED** in the system prompt for duration changes
- System prompt explicitly tells the LLM to use this instead of manual chaining
- Wraps the PHP endpoint in a single tool call

### Updated System Prompt
Rule 4 changed from "increase duration, unassign downstream, auto_place" to "use extend_task_and_replan — it handles everything atomically."

## Impact Identification Algorithm

```
Given task T on station S at time [start, end]:

1. SAME STATION: ∀ task A on S where A.start >= T.end → impacted
2. INTRA-ELEMENT: ∀ task B in T.element where B.sequence > T.sequence → impacted
3. CROSS-ELEMENT: ∀ element E depending on T.element → all tasks in E → impacted
4. CASCADE (BFS): for each impacted task I on station S':
     ∀ task C on S' where C.start >= I.start → impacted

Filter out: completed tasks, in-progress tasks (start <= now < end)
```
