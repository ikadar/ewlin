## Design

### Forme Status Type

```typescript
/** Forme (die-cutting tool) status */
export type FormeStatus =
  | 'none'        // No forme needed (no die-cutting action or assembly element)
  | 'in_stock'    // Forme available in stock
  | 'to_order'    // Forme needs to be ordered
  | 'ordered'     // Forme has been ordered
  | 'delivered';  // Forme has been delivered
```

### Date Fields

```typescript
export interface Element {
  // ... existing fields ...

  // Paper dates
  paperOrderedAt?: string;    // ISO timestamp when status changed to 'ordered'
  paperDeliveredAt?: string;  // ISO timestamp when status changed to 'delivered'

  // BAT dates
  filesReceivedAt?: string;   // ISO timestamp when status changed to 'files_received'
  batSentAt?: string;         // ISO timestamp when status changed to 'bat_sent'
  batApprovedAt?: string;     // ISO timestamp when status changed to 'bat_approved'

  // Forme dates
  formeOrderedAt?: string;    // ISO timestamp when status changed to 'ordered'
  formeDeliveredAt?: string;  // ISO timestamp when status changed to 'delivered'
}
```

### Date Recording Rules

| Field | Status Transition | Date Field Updated |
|-------|-------------------|-------------------|
| Paper | → `ordered` | `paperOrderedAt = now()` |
| Paper | → `delivered` | `paperDeliveredAt = now()` |
| BAT | → `files_received` | `filesReceivedAt = now()` |
| BAT | → `bat_sent` | `batSentAt = now()` |
| BAT | → `bat_approved` | `batApprovedAt = now()` |
| Forme | → `ordered` | `formeOrderedAt = now()` |
| Forme | → `delivered` | `formeDeliveredAt = now()` |

**Note:** Dates are only set on forward transitions. Changing back to an earlier status does NOT clear the date.

### Die-Cutting Action Detection

An element "has die-cutting action" if any of its tasks is assigned to:
1. A station with `categoryId === 'cat-die-cutting'` (if category exists), OR
2. An outsourced provider with `actionType` containing "découpe" or "die-cut"

```typescript
function hasDieCuttingAction(
  element: Element,
  tasks: Task[],
  stations: Station[],
  providers: OutsourcedProvider[]
): boolean {
  return element.taskIds.some(taskId => {
    const task = tasks.find(t => t.id === taskId);
    if (!task) return false;

    if (task.type === 'Internal') {
      const station = stations.find(s => s.id === task.stationId);
      return station?.categoryId === 'cat-die-cutting';
    }

    if (task.type === 'Outsourced') {
      return task.actionType?.toLowerCase().includes('découpe') ||
             task.actionType?.toLowerCase().includes('die-cut');
    }

    return false;
  });
}
```

### Updated Blocking Logic

```typescript
function isElementBlocked(element: Element): boolean {
  const paperReady = ['none', 'in_stock', 'delivered'].includes(element.paperStatus);
  const batReady = ['none', 'bat_approved'].includes(element.batStatus);
  const platesReady = ['none', 'ready'].includes(element.plateStatus);
  const formeReady = ['none', 'in_stock', 'delivered'].includes(element.formeStatus);

  return !paperReady || !batReady || !platesReady || !formeReady;
}
```

### Date Display Format

Dates are displayed as `DD/MM/YYYY` next to the dropdown:

```
Papier:  [Commandé ▾] 15/01/2026
BAT:     [BAT validé ▾] 20/01/2026
Forme:   [Commandée ▾] 18/01/2026
```

Only show date for statuses that have a corresponding date field.
