## Overview

Optimisation des performances du Pick & Place lors du hover sur la grille. Découplage du rendu du ghost (temps-réel) de la validation (throttled) pour éliminer le lag perçu.

**Origine:** Party Mode discussion (2026-01-15) - analyse performance Pick & Place.

## Technical Notes

### Problème actuel

```
MouseMove → handlePickMouseMove → {
  1. setPickHover({ stationId, snappedY })  ← setState = re-render
  2. Throttled validation                    ← 50ms throttle
  3. setPickValidation(result)               ← re-render
}
```

Chaque mouvement de souris déclenche un re-render via `setPickHover`, causant un lag visible sur le ghost.

### Architecture cible

```
FLUX 1: Position (temps-réel, 0ms)
MouseMove → snapToGrid → ghostRef.style.transform = translateY(y)
                        (pas de setState, pas de re-render)

FLUX 2: Validation (throttled, 100ms OK)
MouseMove → Throttle 100ms → Skip si même slot → Validate → Ring color
                                                  (avec CSS transition 100ms)
```

### Implémentation Ghost sans React

```typescript
// Ref pour la position (pas de state)
const ghostPositionRef = useRef({ stationId: '', y: 0 });

// Dans handlePickMouseMove
const snappedY = snapToGrid(Math.max(0, y), pixelsPerHour);
ghostPositionRef.current = { stationId, y: snappedY };

// PickPreview utilise RAF pour lire la ref
useEffect(() => {
  let rafId: number;
  const update = () => {
    if (elementRef.current && ghostPositionRef.current) {
      elementRef.current.style.transform =
        `translateY(${ghostPositionRef.current.y}px)`;
    }
    rafId = requestAnimationFrame(update);
  };
  rafId = requestAnimationFrame(update);
  return () => cancelAnimationFrame(rafId);
}, []);
```

### Early exit validation

```typescript
const lastValidatedY = useRef<number>(0);
const VALIDATION_THRESHOLD = pixelsPerHour / 4; // ~15 min slot

pickValidationThrottleRef.current = setTimeout(() => {
  // Skip si position dans le même slot
  if (Math.abs(snappedY - lastValidatedY.current) < VALIDATION_THRESHOLD) {
    return;
  }
  lastValidatedY.current = snappedY;
  // ... validation coûteuse
}, 100);
```

### CSS Transition Ring

```css
.station-column {
  transition: box-shadow 100ms ease-out, background-color 100ms ease-out;
}
```

## v0.3.58 – Pick & Place Performance Optimization

**Release Date:** 2026-01-16

### Summary

Optimisation significative des performances du Pick & Place. Le ghost suit maintenant le curseur sans aucun lag perçu grâce au découplage du rendu temps-réel de la validation.

### What's New

- Ghost position via manipulation DOM directe (bypass React re-renders)
- Validation throttled à 100ms avec early exit si même slot
- Transition CSS 100ms sur le ring color pour masquer la latence

- Ghost position stockée dans une ref, pas un state
- PickPreview utilise RAF pour lire la position
- Réduction de ~90% des re-renders pendant le hover
- Réduction de ~80% des validations inutiles

### Performance Gains

| Métrique | Avant | Après |
|----------|-------|-------|
| Ghost latency | ~50-100ms | < 16ms |
| Re-renders/sec | ~60 | ~6 |
| Validation calls/sec | ~20 | ~4 |
