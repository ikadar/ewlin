## Technical Notes

### Job Patterns

**1. Single Element (60% of jobs)**
```
Job → Element "ELT" → Tasks
```
No inter-element precedence.

**2. Multi-Sheet (20% of jobs)**
```
Job → Element "F1" → Tasks (parallel)
    → Element "F2" → Tasks (parallel)
    → Element "FIN" → Tasks (prerequisite: F1, F2)
```
FIN waits for all Fx elements.

**3. Brochure (20% of jobs)**
```
Job → Element "COUV" → Tasks (parallel)
    → Element "INT1" → Tasks (parallel)
    → Element "INT2" → Tasks (parallel, optional)
    → Element "FIN" → Tasks (prerequisite: COUV, INT1, INT2)
```
FIN waits for COUV and all INTx elements.

### Element Generation

Each element gets 2-4 tasks (same as before, but scoped to element).
