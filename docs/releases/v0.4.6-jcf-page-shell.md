## Technical Notes

### Architectural Decision: Modal Overlay (not Page Route)

The JCF is implemented as a **full-screen modal overlay** rather than a separate page route:

1. **Scheduling context preserved**: The scheduling grid remains visible behind the semi-transparent backdrop
2. **No router needed**: The main app has no client-side router (React Router). Adding one for a single modal would be over-engineering
3. **Reference pattern**: The JCF reference uses the same modal approach (state 'home' | 'job')
4. **State-based**: Simple boolean `isJcfModalOpen` state in App.tsx

### Modal Structure

```
JcfModal (fixed inset-0, z-50)
├── Backdrop (bg-black/60 backdrop-blur-sm)
└── Dialog (w-[70vw] max-w-[1400px] max-h-[90vh])
    ├── Header (title + close button)
    ├── Content (flex-1 overflow-y-auto, px-6 py-4)
    │   ├── [Job Header area — placeholder for v0.4.7]
    │   └── [Elements Table area — placeholder for v0.4.9]
    └── Footer
        ├── Keyboard hints bar (Tab, Alt+arrows, ↑↓, Cmd+S, Esc)
        └── Action buttons area (placeholder for save buttons)
```

### Integration Points

- `App.tsx`: New state `isJcfModalOpen`, handler `handleOpenJcf` / `handleCloseJcf`
- `JobsList`: Pass `onAddJob={handleOpenJcf}` to enable the "+" button
- `JcfModal`: Receives `isOpen`, `onClose` props
