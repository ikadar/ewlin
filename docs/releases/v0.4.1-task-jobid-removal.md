# Release v0.4.1 – Task.jobId Removal (Element Layer Cleanup)

> **Status:** 🟢 Released

> **Milestone:** M4
>
> **Target Date:** 2025-01-25
>
> **Git Tag:** `v0.4.1`

---

## Overview

This release completes the Element layer integration by removing the deprecated `Task.jobId` property. After v0.4.0 introduced the Element entity with a transitional approach (keeping both `jobId` and `elementId` on Task), this release removes the backward compatibility layer and enforces the clean architecture:

**Before (v0.4.0):** `Task.jobId` + `Task.elementId` (redundant)
**After (v0.4.1):** `Task.elementId` only → `Element.jobId` → `Job`

## Technical Notes

### Helper Functions

The `taskHelpers.ts` utility provides these functions:

```typescript
// Get jobId for a single task
getJobIdForTask(task: Task, elements: Element[]): string | undefined

// Get all tasks for a job
getTasksForJob(jobId: string, tasks: Task[], elements: Element[]): Task[]

// Create a map from taskId to jobId for efficient lookups
createTaskToJobMap(tasks: Task[], elements: Element[]): Map<string, string>

// Group all tasks by their job ID
groupTasksByJob(tasks: Task[], elements: Element[]): Map<string, Task[]>
```

### Component Props Changes

Components that previously used `task.jobId` now need `elements` prop:

- `JobsList` - needs `elements` for conflict job lookup
- `JobDetailsPanel` - needs `elements` for task filtering
- `SchedulingGrid` - needs `elements` for job lookup in tiles

## Release Notes Draft

```markdown
## v0.4.1 – Task.jobId Removal (Element Layer Cleanup)

**Release Date:** 2025-01-25

### Summary

Completes the Element layer integration by removing the deprecated `Task.jobId` property. All task-to-job lookups now use the proper path: `Task.elementId → Element.jobId → Job`.

### What's New

- Clean architecture: Task only references Element, not Job directly
- Helper utilities for task-to-job lookups (`taskHelpers.ts`)
- Components properly pass `elements` for job resolution

### Breaking Changes

- `Task.jobId` no longer exists
- Functions that previously used `task.jobId` now require `elements` parameter

### Migration Guide

Replace direct `task.jobId` access with helper functions:

```typescript
// Before
const jobId = task.jobId;
const jobTasks = tasks.filter(t => t.jobId === jobId);

// After
import { getJobIdForTask, getTasksForJob } from './utils/taskHelpers';
const jobId = getJobIdForTask(task, elements);
const jobTasks = getTasksForJob(jobId, tasks, elements);
```
```
