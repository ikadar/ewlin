## Overview

This release adds proper loading and error state handling to the schedule snapshot fetching. When the app loads data from the real API, users will see a loading indicator during fetch and a retry option if the request fails.

**Key deliverable:** Loading spinner during data fetch and error state with retry button for failed requests.

## Technical Notes

### RTK Query Loading States

The `useGetSnapshotQuery` hook already provides all necessary state:

```typescript
const {
  data: snapshotData,
  isLoading,      // true during first fetch
  isFetching,     // true during any fetch (including refetch)
  isError,
  error,
  refetch
} = useGetSnapshotQuery();
```

### Implementation Approach

1. **Loading State**: Show a centered spinner when `isLoading` is true
2. **Error State**: Show error message with retry button when `isError` is true
3. **Normal State**: Show the existing app content when data is available

### UI Components

- **LoadingSpinner**: Simple CSS spinner (no external dependencies)
- **ErrorState**: Error message card with retry button (French text)
