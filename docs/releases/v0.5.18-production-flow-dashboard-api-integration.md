**Jóváhagyott egyszerűsítések:**

- K3.2 (late állapot definíciója) → halasztva: station state számítás nem implementált

## Technical Notes

### Dual-format `transformResponse`

A `fluxApi.ts` `transformResponse` különbözteti meg a valós API és a mock FluxJob formátumot:
- Valós API: `'shipper' in first` → `FluxJobApiResponse[]` → mapping → `FluxJob[]`
- Mock: nincs `shipper` mező → `FluxJob[]` as-is visszaadva

### RTK Query cache mint igazságforrás

A `FluxPage` nem tart lokális `jobs` state-et — az RTK Query cache a forrás. Prerequisite módosítások és törlések `fluxApi.util.updateQueryData` optimista cache update-tel dolgoznak (eslint-plugin-react-hooks v7 miatt, amely tiltja a `setState in useEffect` mintát).

### Portal-based dropdown

A `FluxPrerequisiteListbox` `createPortal`-t használ a dropdown `document.body`-ra renderelésére, elkerülve a `position: fixed` CSS stacking context interferenciáját (sticky z-20 tábla fejlécek stb.).
