## Overview

Extends the Sequence multi-line autocomplete (v0.4.20) with full ST (Sous-Traitant / Subcontractor) mode. When the user selects "ST:" from the poste suggestions, the autocomplete transitions into a 3-step flow: ST name selection, duration input (with `j`/`h` suffixes), and free-text description. The full format is `ST:Name(duration):description`.

This is the second of three Sequence releases (v0.4.20 Poste mode, v0.4.21 ST mode, v0.4.22 Workflow suggestions).

## Scope

### In Scope

- **ST name suggestions**: After "ST:", show 5 preset sous-traitant names (MCA, F37, LGI, AVN, JF), filtered by typed text
- **ST duration suggestions**: After "ST:Name(", show duration presets with `j`/`h` suffixes (`1j`, `2j`, `3j`, `4j`, `5j`)
- **ST description step**: After "ST:Name(duration):", cursor positioned for free-text description (no autocomplete, just open textarea)
- **ST validation regex**: `/^ST:[A-Za-z0-9_]+\(\d+[jh]?\):.+$/`
- **Default ST durations**: `['1j', '2j', '3j', '4j', '5j']` (days)
- **Session learning for ST names**: Add `learnSoustraitant` to `useSessionLearning`
- **Dropdown keeps open** through ST: → name → duration flow (closes after duration selection with "):")
- **Update sequenceDsl.ts**: Enhance st-name and st-duration parsing, add `DEFAULT_ST_DURATIONS`
- **Update JcfSequenceAutocomplete**: Fill in st-name and st-duration suggestion cases
- **Pass soustraitant presets** from ElementsTable to SequenceAutocomplete
- **Unit tests** for DSL changes and component ST behavior
- **E2E tests** for ST selection flow

### Out of Scope

- Workflow-guided suggestions with priority/star markers (v0.4.22)
- Per-line error highlighting/red background (v0.4.23)
- Custom ST duration formats beyond j/h (e.g., "2s" for semaines)
- ST name descriptions in dropdown (SoustraitantPreset has only `name`)

## Technical Notes

- **3-step ST flow**: ST: → select name (appends `(`) → select duration (appends `):`) → type description (free text, no autocomplete)
- **Duration formats**: `Nj` (days), `Nh` (hours), `N` (plain number) — the `j` and `h` suffixes differentiate from poste durations
- **Default ST durations**: `['1j', '2j', '3j', '4j', '5j']` — distinct from poste `DEFAULT_DURATIONS`
- **Dropdown open/close logic**: Keep open after ST: selection (name step) and after name selection (duration step). Close after duration selection (description is free text).
- **Session learning**: Add `soustraitants` array and `learnSoustraitant` callback to `useSessionLearning`. Session ST names appear first in suggestions.
- **Validation**: ST regex already exists in sequenceDsl.ts: `/^ST:[A-Za-z0-9_]+\(\d+[jh]?\):.+$/`. The description part (`.+`) requires at least 1 character.
- **parseLine enhancement**: The st-name and st-duration steps already exist but have no suggestion generation. This release fills them in.
- **Component prop additions**: Add `soustraitantPresets` and `sessionSoustraitants` props, plus `onLearnSoustraitant` callback.
