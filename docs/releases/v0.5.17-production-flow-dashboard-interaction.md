## Overview

A harmadik lépés a Production Flow Dashboard megvalósításában: az interakciós réteg. A v0.5.15 (statikus táblázat) és v0.5.16 (szűrés + navigáció) alapjaira épülve a prerequisite badge-ek kattintható listbox dropdown-okká válnak, a multi-element sorok expand/collapse-szal nyithatók, a delete és edit akciók valóban működnek. A sub-sor prerequisite módosítása azonnal újraszámolja a parent sor aggregált worst értékét.

## Technical Notes

- **FluxTableContext:** A `FluxTable` belső `useState`-ben tartja az `openListboxId`-t (egyszerre csak egy listbox lehet nyitva). A Context.Provider-en keresztül az összes child komponens (row, sub-row, listbox) eléri a callback-eket és az expanded state-et.
- **Prerequisite listbox koordináció:** Minden listbox egyedi ID-val rendelkezik (`${jobId}-${elementId}-${column}`). Új listbox megnyitásakor az előző automatikusan bezárul (Context értékcsere).
- **Listbox pozícionálás:** `position: fixed` (viewport-relative), `getBoundingClientRect()` a trigger elemről. A táblázat horizontális görgetése nem vágja le a dropdown-t.
- **Click-outside:** `mousedown` esemény figyelő a `document`-en (nem `click`), hogy a bezárás a kiválasztás előtt történjen.
- **Sub-row re-aggregation (K8.1):** A `handleUpdatePrerequisite` immutable update (`setJobs` → map → spread). A parent badge a `FluxTableRow`-ban mindig a jobs state-ből számítja a worst értéket, ezért azonnal frissül.
- **Expand state:** `FluxPage`-ben `Set<string>` state. A `FluxTable` a `flatMap`-ben minden multi-element job után beilleszti a sub-row-okat, ha a job ID a Set-ben van.
- **Animációk:** `flux-subrow-in` (400ms, cubic-bezier, staggered 30ms/elem) és `flux-listbox-in` (150ms, ease-out) keyframe-ek az `index.css`-ben.
- **Delete flow:** `handleDeleteJob` → `deleteConfirmJobId` state → dialog render. `handleConfirmDelete` → `setJobs(filter)` + `setExpandedJobIds(delete)` + clear.
- **Edit flow:** `handleEditJob(jobId)` → `navigate('/job/${jobId}')`.
