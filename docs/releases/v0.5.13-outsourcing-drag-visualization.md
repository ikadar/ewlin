## Overview

Show outsourcing constraints during drag operations, similar to how drying time is visualized. When dragging a task that has an outsourced predecessor or will be followed by an outsourced successor, display the constraint duration visually to help users understand valid drop positions.

## Technical Notes

### When to Show Outsourcing Indicator

Show the indicator when:
1. **Dragging a task after outsourced predecessor**: The successor's earliest start is the outsourced task's return time
2. The outsourced predecessor is scheduled (has an assignment)

### Visual Design

Similar to DryingTimeIndicator but with cyan/teal color to differentiate:
- Vertical arrow from outsourced task's departure to return
- Dashed horizontal line at "earliest start" position
- Label: "Fin outsourcing" (End of outsourcing)

### Integration Point

The indicator should be rendered in the station column where the **successor task** will be placed, showing when it can start after the outsourced predecessor returns.

### Calculation Flow

1. During drag, check if the dragged task has an outsourced predecessor
2. If yes, get the outsourced predecessor's return time (using `getOutsourcedTaskReturnTime` from v0.5.12)
3. Calculate Y position for the return time
4. Show indicator at that Y position
