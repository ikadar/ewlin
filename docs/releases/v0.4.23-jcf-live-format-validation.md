## Overview

Adds Level 1 (Live Format) validation to the JCF elements table. As users type, fields are validated against DSL format rules. Invalid fields show a red background and an error tooltip with helpful guidance. The validation is lenient for incomplete input — errors only appear when the user has finished typing a complete value that doesn't match the expected format.

This is the first of three validation releases (v0.4.23 Level 1 Live, v0.4.24 Level 2 Required, v0.4.25 Level 3 Submit).

## Scope

### In Scope

- **ErrorTooltip component**: Badge with "!" that shows tooltip on hover/focus
- **Validation rules per field type**:
  - `pagination`: Must be 2 (feuillet) or multiple of 4 (cahier)
  - `papier`: Must contain `:` (e.g., "Couché:135")
  - `imposition`: Must contain `(N)` or `(Np)` (e.g., "50x70(8)")
  - `impression`: Must contain `/` (e.g., "Q/Q", "CMJN/")
  - `surfacage`: Must contain `/` (e.g., "mat/mat", "brillant/")
  - `format`: ISO A0-A10, LxH, with optional f/fi suffix, or composite
  - `sequence`: Smart validation (only complete lines are checked)
- **Visual feedback**: Red background (`bg-red-900/20`) on invalid cells
- **Lenient typing**: No error while user is still typing incomplete values
- **Smart sequence validation**: Lines without closing `)` are not flagged
- **Unit tests** for validation logic
- **E2E tests** for validation feedback

### Out of Scope

- Level 2 validation (required field indicators) — v0.4.24
- Level 3 validation (submit blocking) — v0.4.25
- Validation on blur (touched state tracking)
- Validation error summary

---

## Technical Notes

- **Validation timing**: Validate on every keystroke, but only show errors for "complete" values
- **Lenient rules**:
  - `papier`: Error only if value exists AND doesn't contain `:`
  - `imposition`: Error only if value exists AND doesn't match `\(\d+p?\)`
  - `sequence`: Error only if line has closing `)` but doesn't match valid format
- **ErrorTooltip positioning**: Fixed position, calculated to stay within viewport
- **HTML in tooltips**: Support `<strong>`, `<code>`, `<br>` for rich formatting
- **Transition**: Red background fades in over 500ms for smooth appearance
