## Overview

Remplacement du drag & drop depuis la sidebar par un système "Pick & Place" inspiré du peintre qui trempe son pinceau dans la peinture puis le pose sur sa toile. Cette approche résout les problèmes de performance du drag & drop classique en évitant le calcul continu pendant le mouvement de la souris.

**Origine:** Party Mode discussion (2026-01-14) après échec de l'optimisation O(1) des bornes de précédence.

**UX Pattern:**
1. Click sur une tâche non planifiée dans la sidebar → "Pick" (la tâche suit le curseur)
2. La preview apparaît à côté du curseur, affichant la tuile
3. Click sur la grille → "Place" (validation et placement)
4. Click hors grille ou Escape → Annulation

## Technical Notes

### Différence avec le Drag & Drop actuel

```
DRAG & DROP (actuel - performance issues):
  onMouseDown → startDrag()
  onMouseMove (chaque frame) → validate(), updatePreview()
  onMouseUp → drop()

PICK & PLACE (nouvelle approche):
  onClick (sidebar) → pick()
  onMouseMove → updatePreviewPosition() (léger, pas de validation lourde)
  onMouseEnter (station column) → validate() (une fois)
  onClick (grid) → place()
```

### Spécification UX - Ghost Tile (WYSIWYG)

**Principe clé :** Ce que l'utilisateur voit = ce qu'il obtient (What You See Is What You Get)

```
Quand l'utilisateur survole une station COMPATIBLE :

    Station Column
    ┌─────────────┐
    │             │
    │ ┌─────────┐ │ ← Position snappée (15 min)
    │ │░░GHOST░░│ │
    │ │░░TILE░░░│ │    Hauteur = taille EFFECTIVE
    │ │░░░░░░░░░│ │    (incluant étirement working hours)
    │ └─────────┘ │
    │      ✕      │ ← Curseur quelque part dans la colonne
    │             │
    └─────────────┘

Quand l'utilisateur survole une station INCOMPATIBLE :
- Pas de ghost tile
- Curseur "not-allowed"
```

**Positionnement de la Ghost Tile :**
- Position X : Alignée dans la station column (pas à côté du curseur)
- Position Y : Snappée aux intervalles de 15 minutes
- Hauteur : Calculée avec `calculateEndTime()` (prend en compte working hours)

**Pas de décalage :** La ghost tile est affichée exactement là où la tuile sera placée au clic.

### Réutilisation du Quick Placement Mode

Le Quick Placement Mode (v0.3.16) offre déjà:
- Indicateur de position (ligne blanche au snap)
- Curseur interdit quand pas de tâche
- Validation au clic
- Lignes de précédence

Différences avec Pick & Place:
- Quick Placement: sélection automatique de la prochaine tâche du job
- Pick & Place: sélection explicite d'une tâche spécifique

### States

```typescript
interface PickState {
  /** Whether pick mode is active */
  isPicking: boolean;
  /** The task being picked (null when not picking) */
  pickedTask: Task | null;
  /** The job the picked task belongs to */
  pickedJob: Job | null;
  /** Current cursor position (for preview) */
  cursorPosition: { x: number; y: number } | null;
  /** Target station (when hovering over grid) */
  targetStationId: string | null;
  /** Validation result at current position */
  validation: DropValidationResult | null;
}
```

## Comparison: Drag vs Pick & Place

| Aspect | Drag & Drop | Pick & Place |
|--------|-------------|--------------|
| Initiation | MouseDown + Move | Single Click |
| Preview | Follows cursor during drag | Follows cursor after pick |
| Validation | Every frame | On station enter/exit |
| Cancel | Drop outside / ESC | Click outside / ESC |
| Performance | O(n) per frame | O(1) most of time, O(n) on station change |
| UX Mental Model | "Tenir et déplacer" | "Tremper et poser" |
