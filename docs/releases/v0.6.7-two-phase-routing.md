# v0.6.7 - Two-Phase Intent Routing

**Status:** In Progress
**Branch:** `feature/mcp-llm-scheduler`
**Date:** 2026-03-29

## Summary

Reduces tool definition tokens by ~87% per call via two-phase routing: a cheap classification call first (no tools, ~160 tokens), then only the relevant tool subset is sent.

## How It Works

### Phase 1: Classification (~160 tokens)
A minimal API call with no tools asks the LLM to classify the user message into one of 10 intent categories. Uses the same model (Haiku by default). Cost: ~$0.00013 per classification.

### Phase 2: Targeted Tool Call
Based on the classified intent, only 1-7 relevant tools are sent instead of all 27.

| Intent | Tools | Count |
|--------|-------|-------|
| lookup | schedule summary, station lookup, job search | 3 |
| duration | job search, station lookup, extend+replan, update duration | 4 |
| completion | job search, complete tasks, complete before time | 3 |
| placement | job search, station lookup, auto-place (global/ASAP/ALAP) | 5 |
| station | station lookup, add exception | 2 |
| job_update | job search, update/delete job, dependencies, outsourced status | 7 |
| task_ops | job search, station lookup, assign/reschedule/split/fuse/unassign | 7 |
| prerequisite | job search, update prerequisite | 2 |
| schedule_mgmt | list/save/load schedules | 3 |
| compact | smart compact | 1 |

**Average: 3.7 tools/call instead of 27 = 87% reduction**

If classification fails, falls back to all 27 tools.

## Token Impact

| Before | After |
|--------|-------|
| ~1,900 tokens in tool defs/call | ~260 tokens avg + 160 classification |
| | **~78% net reduction** |

Combined with prompt caching (v0.6.4), the total static overhead per call is now ~500 tokens (was ~3,800 before all optimizations).
