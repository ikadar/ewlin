## Overview

Provides mock reference data and mock API endpoints for all JCF autocomplete fields. This release enables upcoming UI releases (v0.4.6+) to query paper types, product formats, imposition sheet formats, printing presets, surfacage presets, machine presets, and subcontractor lists from an in-memory mock API layer.

The data is modeled after the reference JCF's `data/*.json` files but reimplemented as TypeScript modules using the `@flux/types` interfaces defined in v0.4.4.

## Technical Notes

### Architecture Decision: Static Data Module

Reference data is static (rarely changes in production). For the mock layer, we store it as typed constant arrays rather than generating it dynamically. This matches the reference JCF pattern (JSON files loaded by storage service) while using `@flux/types` interfaces for type safety.

```typescript
// Pattern: Static typed constant
import type { PaperType } from '@flux/types';

export const PAPER_TYPES: PaperType[] = [
  { id: 'couche-mat', type: 'Couché mat', grammages: [70, 80, 90, ...] },
  // ...
];
```

### Data Differences from Reference JCF

| Field | Reference JCF | Ordo (v0.4.5) |
|-------|--------------|---------------|
| `PaperType.grammages` | `string[]` ("70g") | `number[]` (70) — per v0.4.4 type def |
| `PostePreset` | Has `id` field | No `id` — keyed by `name` per v0.4.4 |
| `SoustraitantPreset` | Has `id` field | No `id` — keyed by `name` per v0.4.4 |

### Mock API Extension Pattern

New methods are added to the existing `MockApi` interface:

```typescript
export interface MockApi {
  // ... existing methods ...

  // Reference data endpoints (v0.4.5)
  getPaperTypes(): Promise<PaperType[]>;
  getProductFormats(): Promise<ProductFormat[]>;
  getFeuilleFormats(): Promise<FeuilleFormat[]>;
  getImpressionPresets(): Promise<ImpressionPreset[]>;
  getSurfacagePresets(): Promise<SurfacagePreset[]>;
  getPostePresets(): Promise<PostePreset[]>;
  getSoustraitantPresets(): Promise<SoustraitantPreset[]>;
  getTemplateCategories(): Promise<TemplateCategory[]>;
}
```
