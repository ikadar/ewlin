## Overview

Fixes a bug where tile snapping works correctly at 100% zoom level but breaks at other zoom levels. The root cause is that several code locations call `snapToGrid` and `yPositionToTime` without passing the current `pixelsPerHour` value, causing them to use the default 80px/hour instead of the zoom-adjusted value.

## Technical Notes

### Root Cause Analysis

The `snapToGrid` and `yPositionToTime` functions in `snapUtils.ts` already support a `pixelsPerHour` parameter with a default value of `PIXELS_PER_HOUR` (80). However, several call sites don't pass the current zoom-adjusted value:

**Affected code:**

| File | Line | Issue |
|------|------|-------|
| `DragLayer.tsx` | 74 | `snapToGrid(contentY)` - missing pixelsPerHour |
| `DragLayer.tsx` | 79 | `snapToGrid(position.y - grabOffset.y)` - missing pixelsPerHour |
| `App.tsx` | ~1130 | `handleQuickPlacementMouseMove` - missing pixelsPerHour |
| `App.tsx` | ~1157-1158 | `handleQuickPlacementClick` - missing pixelsPerHour for both functions |

### Solution Approach

1. **Extend DragStateContext** to include `pixelsPerHour` value
2. **Update DragLayer.tsx** to read pixelsPerHour from context
3. **Update Quick Placement handlers** in App.tsx to pass pixelsPerHour
