## Overview

Updates the precedence constraint calculation to handle inter-element dependencies. The first task of an element with prerequisites must wait for all prerequisite elements to complete.

## Technical Notes

### Precedence Rules

**Intra-element (existing):**
- Task N+1 cannot start before Task N ends
- Applies to all tasks within an element

**Inter-element (new):**
- First task of element E cannot start before all prerequisite elements complete
- "Complete" = last task of prerequisite element ends
- Applies ONLY to first task of the dependent element

### Bound Calculation

```typescript
// For first task of element with prerequisites
const interElementBound = Math.max(
  ...element.prerequisiteElementIds.map(prereqId => {
    const prereqElement = elements.find(e => e.id === prereqId);
    const lastTaskId = prereqElement.taskIds[prereqElement.taskIds.length - 1];
    const lastTaskAssignment = assignments.find(a => a.taskId === lastTaskId);
    return lastTaskAssignment ? new Date(lastTaskAssignment.scheduledEnd).getTime() : 0;
  })
);

// For any task, intra-element bound
const intraElementBound = previousTaskInElement
  ? new Date(previousTaskAssignment.scheduledEnd).getTime()
  : 0;

// Final bound = MAX of both
const finalBound = Math.max(interElementBound, intraElementBound);
```

### Example

```
Job: Brochure
├── COUV: T1 → T2 → T3 (T3 ends at 14:00)
├── INT:  T1 → T2      (T2 ends at 12:00)
└── FIN:  T1 → T2      (prerequisite: COUV, INT)

FIN.T1 bound = MAX(14:00, 12:00) = 14:00
FIN.T2 bound = end of FIN.T1 (intra-element only)
```
