## Overview

Add React Router to enable URL-based navigation and deep linking. Users will be able to bookmark and share links to specific views, and the browser's back/forward buttons will work naturally with the application state.

## Technical Notes

### Route Structure

```
/                    → Main scheduler view (no job selected)
/job/:jobId          → Main scheduler with job selected
/job/new             → Main scheduler with JCF modal open
```

### Implementation Strategy

1. Wrap app with `BrowserRouter` in main.tsx
2. Define routes in a new `Router.tsx` component
3. Replace `selectedJobId` state navigation with React Router
4. Use `useParams()` to read jobId from URL
5. Use `useNavigate()` to update URL on job selection
6. Handle JCF modal via `/job/new` route

### State Synchronization

- **Job Selection**: URL is source of truth
  - Click job → `navigate(/job/${jobId})`
  - URL change → update Redux `selectedJobId`
- **JCF Modal**: Route-based
  - Click "+" button → `navigate('/job/new')`
  - Close modal → `navigate('/')` or `navigate(-1)`
