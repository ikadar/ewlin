## Overview

Implements the JCF Elements Table — a row-based CSS Grid layout where rows represent field types (precedences, quantité, format, etc.) and columns represent elements. Includes element header with name editing (click to rename), add/remove element buttons, and plain text inputs for all 12 field rows + sequence row. This release establishes the table structure and data model; autocomplete fields, cell navigation, validation, and links are added in later releases.

## Scope

### In Scope

- Row-based CSS Grid layout (`gridTemplateColumns: 100px repeat(N, 288px)`)
- 12 base field rows: precedences, quantité, pagination, format, papier, impression, surfacage, autres, imposition, qteFeuilles, commentaires
- Sequence row (multi-line textarea, separate treatment)
- Sticky left column for row labels
- Element header row with element name display
- Element name editing — click to enter edit mode, Enter/blur to save, Escape to cancel
- Element name generation — `ELEM1`, `ELEM2`, etc.
- Add element button (+) — inserts after the clicked element
- Remove element button (−) — disabled when only 1 element remains
- Default element: `{ name: 'ELT', precedences: '', quantite: '1', ...empty fields }`
- Plain text `<input>` for all fields (autocomplete components added in later releases)
- Commentaires field as auto-expanding `<textarea>`
- Numeric alignment (`text-right`) for quantite, pagination, qteFeuilles
- Precedences row visual distinction (dimmed styling, git-branch icon)
- Integration into JcfModal (rendered below JcfJobHeader)
- State management: `elements` array lifted to App.tsx
- rem→px conversion for 13px base (all Tailwind classes converted)

### Out of Scope

- Cell navigation — Tab, Shift+Tab, Alt+Arrow (v0.4.10)
- Autocomplete fields for any row (v0.4.11–v0.4.22)
- Field linking system (v0.4.15–v0.4.16)
- Validation system (v0.4.23–v0.4.25)
- Auto-calculated qteFeuilles (v0.4.23+)
- Tooltips on hover (comes with validation)
- Precedences auto-update on rename (comes with precedences autocomplete v0.4.19)
- Save functionality (v0.4.26)

## Feature Checklist

### Data Model: Element interface (local)

- [ ] **JCF Element type** — local interface for JCF form state: `{ name, precedences, quantite, format, pagination, papier, imposition, impression, surfacage, autres, qteFeuilles, commentaires, sequence }` — all string fields
- [ ] **Default element** — `{ name: 'ELT', quantite: '1', all others empty string }`
- [ ] **Row definitions** — `baseRows` array of `{ key, label, icon? }` for 11 base fields + sequence row

### Component: JcfElementsTable

- [ ] **Grid layout** — CSS Grid with `gridTemplateColumns: 100px repeat(N, 288→px equivalent)` (13px base: `288px` = reference `w-72` = 18rem × 16 = 288px — note: reference uses 16px base for grid columns)
- [ ] **Outer container** — `bg-zinc-900/30 rounded border border-zinc-800 overflow-x-auto w-fit max-w-full`
- [ ] **Header row** — grid row with empty label cell + element name headers
- [ ] **Sticky label column** — `sticky left-0` with z-index, background color for opacity
- [ ] **Field rows** — 12 rows (11 base + sequence), each a grid row with label + data cells
- [ ] **Row separator** — `border-b border-zinc-800/50` between rows
- [ ] **Column separator** — `border-r border-zinc-800/50` between element cells (except last)
- [ ] **Hover effect** — `hover:bg-zinc-900/30` on non-precedences rows
- [ ] **Precedences row styling** — dimmed: `bg-zinc-900/50`, `text-zinc-600`, git-branch icon (Lucide)

### Component: Element Header

- [ ] **Element name display** — clickable text, hover: `bg-zinc-800/50` rounded, cursor-pointer
- [ ] **Element name editing** — click to edit: inline `<input>` with `autoFocus`, `onFocus` selects all
- [ ] **Name save** — Enter key or blur saves; empty name cancels edit
- [ ] **Name cancel** — Escape key cancels edit, restores original name
- [ ] **Name generation** — `generateElementName()`: scans existing names matching `ELEM{N}`, returns `ELEM{max+1}`
- [ ] **Add button** — `+` icon (Lucide Plus), inserts new element after current
- [ ] **Remove button** — `−` icon (Lucide Minus), removes current element; disabled when only 1 element
- [ ] **Button styling** — zinc-600 default; hover: green/blue for add, red for remove; disabled: opacity-30, cursor-not-allowed

### Component: Data Cells (plain text inputs)

- [ ] **Standard input** — transparent bg, border on hover/focus, `font-mono`, `text-zinc-100`
- [ ] **Empty input** — same as standard but `text-zinc-500`
- [ ] **Numeric alignment** — `text-right` for quantite, pagination, qteFeuilles rows
- [ ] **Commentaires textarea** — auto-expanding `<textarea>` with `resize-none`, `min-h-[28px→px equiv]`, `overflow-hidden`
- [ ] **Sequence textarea** — multi-line `<textarea>`, same auto-expand behavior
- [ ] **Cell change handler** — updates `elements[index][field]` in state

### Integration: App.tsx

- [ ] **elements state** — `useState<JcfElement[]>` with default element
- [ ] **Reset on modal close** — reset elements to default on `handleCloseJcf`
- [ ] **Pass to JcfElementsTable** — `elements`, `onElementsChange` props
- [ ] **Render in JcfModal** — below JcfJobHeader

### Barrel exports

- [ ] **JcfElementsTable/index.ts** — export component + types
- [ ] **components/index.ts** — add JcfElementsTable export

### rem→px conversion (13px base)

- [ ] **Grid column width** — reference uses `288px` literally (not rem-based), keep as-is
- [ ] **Label column** — reference `100px` literally, keep as-is
- [ ] **Padding px-3** — 0.75rem × 13 = `px-[10px]`
- [ ] **Padding py-2** — 0.5rem × 13 = `py-[7px]`

- [ ] **Padding px-1** — 0.25rem × 13 = `px-[3px]`
- [ ] **Text text-sm** — 0.875rem × 13 = `text-[11px]`
- [ ] **Icon w-3.5 h-3.5** — 0.875rem × 13 = `w-[11px] h-[11px]`
