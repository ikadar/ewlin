## Overview

Station category columns (Off., Mass., Pell., etc.) are now sortable. Clicking a station column header sorts the table by that column's worst-state severity across all elements. This closes deferred issue K9.2 and fully implements spec 3.6 for all columns.

---

## Prerequisites

- [x] `v0.5.23` released (ST column frontend UI)

---

## Scope

### In Scope

- `SortColumn` type extended with `station:${string}` template literal
- `worstStationSeverity()` helper: worst state across all elements of a job at a given station category
- Station sorting branch in `sortFluxJobs()`: ascending = best (done/empty) first; descending = worst (late) first
- Station column headers clickable with `SortChevron` indicator (hover + active state)
- Unit tests for all station sorting cases

### Out of Scope

- Progress % sorting (K3.1) — separate release
- ST column sorting — not sortable per spec
- Parti column sorting (K5.1 still deferred)
- URL persistence for sort state

## Technical Notes

- Severity: `late(0) > in-progress(1) > planned(2) > done(3) > empty(4)` — reuses `STATION_STATE_SEVERITY` from `fluxAggregation.ts`
- Multi-element jobs: worst state across all elements (same pattern as prerequisite columns)
- Template literal `station:${categoryId}` distinguishes station columns from other `SortColumn` values
- No changes needed in `FluxPage` or context — `onSortChange`, `sortColumn`, `sortDirection` are already props
