## Overview

Implement the complete template management system for the Job Creation Form (JCF). This release enables users to:
- Create templates from existing jobs ("Save as template")
- Apply templates to new jobs (populate elements table)
- View, search, edit, and delete templates in a dedicated list view

## Technical Notes

### Template Type Definition

```typescript
// packages/types/src/template.ts
export interface JcfTemplate {
  id: string;
  name: string;
  description?: string;
  category?: string;
  clientName?: string;  // null = universal template
  elements: JcfTemplateElement[];
  createdAt: string;  // ISO 8601
  updatedAt: string;  // ISO 8601
}

export interface JcfTemplateElement {
  name: string;
  precedences: string;
  quantite: string;
  format: string;
  pagination: string;
  papier: string;
  imposition: string;
  impression: string;
  surfacage: string;
  autres: string;
  qteFeuilles: string;
  commentaires: string;
  sequence: string;
  sequenceWorkflow?: string[];
  // links?: Partial<Record<JcfFieldKey, boolean>>;  // v0.4.35
}
```

### LocalStorage Persistence

Templates stored in `localStorage` under key `flux-jcf-templates`. Mock API functions:
- `getTemplates(): JcfTemplate[]`
- `getTemplate(id: string): JcfTemplate | null`
- `createTemplate(data: Omit<JcfTemplate, 'id' | 'createdAt' | 'updatedAt'>): JcfTemplate`
- `updateTemplate(id: string, data: Partial<JcfTemplate>): JcfTemplate`
- `deleteTemplate(id: string): void`
