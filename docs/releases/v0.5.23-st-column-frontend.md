## Overview

Adds the ST (Sous-traitance / Subcontracting) column to the Flux dashboard frontend.
Renders 3-state interactive checkboxes (pending / in progress / done) with a custom tooltip,
wires the `PATCH /api/v1/flux/tasks/{taskId}/status` backend endpoint, and adds a new
"S-T à faire" tab to the filter bar.

This is the frontend half of the ST column feature. The backend was delivered in v0.5.22.

## Scope

### In Scope

- `fluxTypes.ts`: new `FluxSTStatus` type, `FluxOutsourcingTask` interface, extend `FluxElement` with `outsourcing: FluxOutsourcingTask[]`
- `fluxApi.ts`: extend `transformFluxJobsResponse` to map `outsourcing[]` from API; add `updateSTStatus` RTK Query mutation (`PATCH /flux/tasks/{taskId}/status`)
- `fluxFilters.ts`: add `'soustraitance'` tab ID, filter logic, URL path `/flux/soustraitance`, count badge
- `fluxStaticData.ts`: add `outsourcing[]` arrays to all 5 reference jobs (spec §3.3 reference data)
- New `STCell.tsx`: 3-state SVG icon, click cycle, "ProviderName · ActionType" label, custom fixed-position tooltip
- `FluxTable.tsx`: ST column header (`<th>` "ST", `title="Sous-traitance"`, not sortable, ~160px), `<STCell>` in parent rows (collapsed multi: flattened tasks) and in sub-rows (per-element tasks)
- `FluxPage.tsx`: `handleUpdateSTStatus` callback passed to table
- `index.css`: `.st-pending`, `.st-progress`, `.st-done`, `.st-tooltip` CSS
- Unit tests: `fluxFilters.test.ts` (soustraitance tab filter — 3/5 jobs visible), `STCell.test.tsx` (3-state render, click cycle)
- E2E tests: `st-column.spec.ts` (click cycle, tooltip visibility)

### Out of Scope

- Optimistic UI update (RTK mutation with `invalidatesTags` is sufficient)
- Offline / error state for the PATCH mutation
- Sorting by ST column (consistent with station columns — not sortable)

## Technical Notes

### Column position

ST column inserted at position 19 (between last station column "Cond." and "Transporteur").

### SVG icons

All 3 SVGs share: `viewBox="0 0 24 24"`, `class="w-4 h-4"`, `fill="none"`, `stroke="currentColor"`, `stroke-width="2"`.

| State | SVG content |
|-------|------------|
| `pending` | `<circle cx="12" cy="12" r="10">` |
| `progress` | `<circle cx="12" cy="12" r="10">` + `<circle cx="12" cy="12" r="4" fill="currentColor" stroke="none">` |
| `done` | `<path d="M22 11.08V12a10 10 0 1 1-5.93-9.14">` + `<path d="m9 11 3 3L22 4">` |

### Click cycle

```
pending → progress → done → pending → ...
```

On click: call `onUpdateSTStatus(taskId, nextStatus)` → RTK mutation PATCH.

### Tooltip

Non-native custom tooltip (`div.st-tooltip`) positioned `fixed`, triggered by `onMouseEnter`/`onMouseMove`/`onMouseLeave` on each ST line. Contains full untruncated "ProviderName · ActionType" text.

### Multi-element collapsed row

All element `outsourcing[]` arrays are **flattened** into one list (`job.elements.flatMap(e => e.outsourcing)`).

### Multi-element expanded sub-rows

Each sub-row renders only its own `element.outsourcing[]`.

### "S-T à faire" tab filter

```typescript
case 'soustraitance':
  return job.elements.some(el =>
    el.outsourcing.some(t => t.status !== 'done')
  );
```

Job is included if at least one task has `status !== 'done'` (pending OR progress).
