### State Location Decision

| Adat típus | Tárolás helye | Hozzáférés |
|------------|---------------|------------|
| Snapshot (jobs, tasks, etc.) | RTK Query cache | `useGetSnapshotQuery()` |
| UI state | `uiSlice` | `useAppSelector(state => state.ui)` |
| JCF form state | `jcfSlice` | `useAppSelector(state => state.jcf)` |

**Döntés:** Snapshot adat az RTK Query cache-ben marad (nincs külön `snapshotSlice`).
- Automatikus cache management
- `isLoading`, `error`, `refetch` ingyen
- M5-ben könnyű átállás valódi API-ra

### RTK Query + Mock Adapter Pattern (Opció A)

A fixture/mock mechanizmus változatlan marad. Az RTK Query `queryFn` hívja a meglévő mock függvényeket:

```typescript
// store/api/scheduleApi.ts
import { createApi } from '@reduxjs/toolkit/query/react';
import { getSnapshot, updateSnapshot } from '../../mock/snapshot';

export const scheduleApi = createApi({
  reducerPath: 'scheduleApi',
  baseQuery: async () => ({ data: null }), // No-op, queryFn használunk
  tagTypes: ['Snapshot'],
  endpoints: (builder) => ({
    getSnapshot: builder.query<ScheduleSnapshot, void>({
      queryFn: () => {
        // Meglévő mock/fixture logika - URL ?fixture= támogatással
        const snapshot = getSnapshot();
        return { data: snapshot };
      },
      providesTags: ['Snapshot'],
    }),
    // További endpoints: createAssignment, updateAssignment, etc.
  }),
});

export const { useGetSnapshotQuery } = scheduleApi;
```

**Előnyök:**
1. Fixture mechanizmus változatlan (`mock/snapshot.ts`, `mock/testFixtures.ts`)
2. Komponensek `useGetSnapshotQuery()` hook-ot használnak
3. M5-ben csak a `queryFn`-t cseréljük valódi `fetch()` hívásra
4. Cache invalidation és refetch automatikusan működik

### Migration Strategy

1. **Phase 1**: Create store infrastructure (store, typed hooks, Provider)
2. **Phase 2**: Add RTK Query API slice with mock adapter (`scheduleApi`)
3. **Phase 3**: Create UI and JCF slices
4. **Phase 4**: Migrate App.tsx to use Redux (selectors + dispatch)
5. **Phase 5**: Update child components to use Redux where needed
6. **Phase 6**: Verify all functionality and E2E tests pass
