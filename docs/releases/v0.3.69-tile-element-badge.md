## Overview

Adds element suffix badge to tiles in the scheduling grid. This allows users to quickly identify which element a task belongs to without selecting the job.

---

## Prerequisites

- [ ] `v0.3.67` released (Element mock generators)

---

## Scope

### In Scope

- Element suffix badge on Tile component
- Badge styling: semi-opaque black background, small border radius
- Badge position: right of job reference
- Badge hidden for single-element jobs (only show for multi-element jobs)

### Out of Scope

- JobDetailsPanel changes (v0.3.68)
- Precedence logic changes (v0.3.70)

## Technical Notes

### Visual Design

```
┌─────────────────────────┐
│ 2026-0042        [couv] │
│ █████████████           │
└─────────────────────────┘
```

### Badge Styling (Tailwind)

```tsx
<span className="bg-black/50 rounded-sm px-1 text-xs">
  {element.suffix}
</span>
```

- `bg-black/50`: Semi-opaque black background
- `rounded-sm`: Small border radius
- `px-1`: Horizontal padding
- `text-xs`: Small text
- Text color: inherit from card title (no explicit color)

### Data Flow

1. Tile receives `task` prop
2. Look up element via `task.elementId`
3. Display `element.suffix` in badge
